# 自由形式のCSVファイル

今のところ、テーブル名を決めて、含む列名の一覧をハードコーディングする、  
みたいになっているCSVファイルのフォーマットですが、  
最終的には、ハードコーディングをいっさい止めたい。  

じゃあ、どうするかというと、  
CSVファイルのテーブル名も、その列名の一覧も、列の値の型も、  
全部、外部ファイルで設定できるようにしたい。  

そんなこと言っても、ハードコーディングしたい［テーブル名］や［列名］は、あると思う。  

つまり、［CSVファイルを利用する側］の数だけ、ハードコーディングしたい。  

* 自由形式という基盤の仕様
* 自由形式を利用した、ハードコーディングしたい部分の仕様

この２つの層に分けて、仕様を考えていきたい。  

`Data\Definitions\`  

というフォルダーを作った。  
ここに［自由形式という基盤の仕様］を定義するファイルを置いていきたい。  

例えば、型は int, double, string の３種類だけでいいと思う。  
bool は要らない。あとで３値、４値にしたくなるかもしれないから、 int で表現する。これは珍しくない、人間がよく使うパターン。  


## テーブル定義 JSON ファイル

テーブル定義を書いておく。  

📄 `StandardRuleTableType.json`:  

```json
{
	"Meta": {
		"Name": "StandardRule"
		"Type": "TableType",
	},
	"Data": {
		{"Name": "name", "Type": "string", "Comment": "名前"},
		{"Name": "age", "Type": "int", "Comment": "年齢"},
		{"Name": "color", "Type": "string", "Comment": "色"}
	}
}
```


## メタ JSON ファイル

📄 `Banana.csv`  
📄 `Banana.json`  

のように、CSVファイルと同じ名前のJSONファイルを用意して、  
`Banana.json` の中に、CSVファイルのテーブル名や列名の一覧を定義する。  
メタデータを色々書けるのが JSON 形式のメリット。  


```json
{
	"Meta": {
		"TableName": "Banana",
		"TableType": "StandardRule"
	}
}
```
