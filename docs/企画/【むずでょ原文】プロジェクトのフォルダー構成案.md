# 【むずでょ原文】プロジェクトのフォルダー構成案

## 互換性用フォルダー

AIエージェントは互換性のロジックを残したがるので、それ用のフォルダーを置いておいてやりたい（＾～＾）　（AIエージェントの飼い方）  

```plaintext
📁 {プロジェクトフォルダー}
+-- 📁 Compatibility
	+-- 📁 {Legacy*}
	+-- 📁 {Internal*}
	*-- 📄 *Facade.cs
```
* `Legacy`：古いコードを残すためのフォルダー。将来的に削除する予定のコードをここに移動する。
* `Internal`：内部で使用するコードを格納するフォルダー。外部に公開しないコードをここに配置する。
* `Facade.cs`：プロジェクトの主要なインターフェースを提供するファサードクラス。外部からのアクセスをここで管理する。

### アクセス規約

`Compatibility` フォルダーの外から参照してよいのは、原則として `Compatibility` 直下のファサードだけとする。

`Compatibility` より深いフォルダーに置いたコードは互換性対応の内部実装として扱い、`Application`、`Domain`、`Infrastructure` などから直接参照しない。

```plaintext
OK:
Application
  -> Compatibility/*Facade.cs

NG:
Application
  -> Compatibility/Legacy*/...
Application
  -> Compatibility/Internal*/...
```

### 命名規約

* `Legacy*`：古い仕様、古いファイル形式、古い入力値など、過去との互換性を取るためのコード。
* `Internal*`：古い仕様そのものではないが、互換性ファサードを実現するための内部実装コード。
* `Private*`：C# の `private` と紛らわしいため、フォルダー名としては原則使わない。

互換性コードは「古い形式を現在の正規の形へ寄せる」ための場所とし、新しい本筋の仕様やドメインロジックをここへ増やさない。
