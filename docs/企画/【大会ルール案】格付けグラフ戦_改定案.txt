【大会ルール案】格付けグラフ戦_改定案

作成日
======
2026-05-14

目的
====
元の「格付けグラフ戦」案は、発想としてはおもしろいが、
そのままでは順位が定義不能になったり、運営が破綻しやすい点がある。  
本改定案では、

- 何を記録するか
- 何をもって「つながった」とするか
- 循環や双方向が起きたときにどう扱うか
- 最終順位をどう確定するか

を明確にして、実際に運営できる形へ寄せる。

基本思想
========
この案の核は、
**「勝敗結果をそのままグラフにして、対局不足の部分を埋めながら序列を作る」**
ことである。

ただし、グラフだけでは順位が一意に決まらないことがあるため、
本改定案では

1. 対局組み合わせを決めるためのグラフ
2. 最終順位を確定するための補助規則

を分けて考える。

改定案の全体像
============
（１）各対局が終わったら、勝者と敗者を記録する。  
先手負けだけではなく、**すべての対局結果** を使う。  

（２）ホワイトボードまたは一覧表に、
**敗者 → 勝者** の矢印を記録する。  

例:
- 自分がいぬに負けたら `自分 → いぬ`
- ぞうがねこに負けたら `ぞう → ねこ`

（３）同じ組み合わせの再戦は、原則として避ける。  
ただし、どうしても必要なら例外を認める。  

（４）次ラウンドの組み合わせは、
**まだ比較関係が薄い者どうし** を優先して当てる。  
特に、別々のグラフ成分に属する者どうしを優先する。  

（５）グラフが増えていったら、
最終順位は次の手順で求める。  

改定ルール詳細
==============
第1条　記録する勝敗
------------------
1. すべての公式対局を記録対象とする。  
2. 先手負けだけを特別扱いしない。  
3. 1局ごとに、敗者から勝者へ有向辺を引く。  
4. 1人が複数回負けた場合は、矢印を複数持ってよい。上書きしない。  

理由:
- 先手負けだけを使うと情報が半分以上欠ける
- 色の偏りで歪む
- グラフが不自然になる

第2条　グラフの意味
------------------
1. 矢印 `A → B` は、「A は B に負けた」を表す。  
2. 「つながっている」とは、まずは **無向に見たときに同じ連結成分にいること** とする。  
3. 対局決定では、この「無向連結」を使う。  

理由:
- 有向の強連結や到達可能性まで使うと運営が複雑になりすぎる
- 組み合わせ決定では、まず「情報が行き来しているか」だけ見ればよい

第3条　対局組み合わせの優先順位
------------------------------
次ラウンドの組み合わせは、次の優先順位で決める。  

第1優先:
- 別々の連結成分にいる者どうし

第2優先:
- 同じ連結成分にいるが、まだ直接対戦していない者どうし

第3優先:
- すでに比較情報が多い者どうしは後回し

補足:
- 先後の偏りが大きくならないように、組み合わせ候補の中で先後バランスを調整する
- 再戦は原則禁止。ただし人数やラウンド数の都合で避けられない場合のみ例外

第4条　循環と双方向
------------------
1. `A → B` と `B → A` のような双方向や、
   `A → B → C → A` のような循環は起こりうる。  
2. 循環が起きた場合、その部分だけで上下は決めない。  
3. 循環している集団は、
   **同一グループ候補** としてまとめて扱う。  

理由:
- 循環がある時点で、単純な「矢印を向けられた方が上」では矛盾する
- そのため、循環部分は一旦まとめる必要がある

第5条　最終順位の決め方
----------------------
最終順位は、次の 3 段階で決める。  

第1段階:
- 強連結成分ごとに 1 つのまとまりとして縮約する
- すると、成分どうしの関係は DAG（閉路のないグラフ）になる

第2段階:
- 縮約後の DAG で、上位層から順に順位帯を作る
- 具体的には、入次数 0 の成分を最上位帯、その次を次順位帯、という形でもよい
- または「他成分から多く指されている側を上位」とする層分けでもよい

第3段階:
- 同じ順位帯の中では、勝ち星で並べる
- それでも同じなら同順位とする

ここで大事なのは、
**グラフだけで無理に全順序を作らない** ことである。  
まず層を作り、その中を勝ち星で整理する。

第6条　実務上の簡易版
--------------------
もし強連結成分や縮約DAGが難しければ、運営上は次の簡易版でもよい。  

簡易版順位決定:
1. 連結成分ごとにまとめる
2. その成分の中では勝ち星順
3. 成分どうしの上下は、成分間にある矢印の向きで判断する
4. 判断しきれないときは同順位扱い

これは厳密さは少し落ちるが、現場運用はかなりしやすい。

元の案との違い
==============
元の案では:
- 先手負けしか記録しなかった
- 「つながる」の意味が曖昧だった
- 双方向や循環が起きたときに順位が壊れた
- 「同じ順位」の定義がなかった

改定案では:
- すべての公式対局を記録する
- 無向連結で対局優先を決める
- 強連結成分を使って循環を処理する
- 最後は層分け + 勝ち星で順位を確定する

これにより、少なくとも
**順位が定義不能になる欠陥** はかなり減る。

それでも残る弱点
================
1. 運営がやや難しい  
   - グラフ更新が必要
   - 強連結成分の把握は手作業では難しい

2. 勝ち星ルールとの二重構造になる  
   - 結局、最後は勝ち星も使うので、グラフの寄与が直感的に見えにくい

3. ラウンド数が少ないと情報不足になる  
   - グラフが疎なまま終わると、比較不能が多く残る

4. 偶然の番狂わせに弱い  
   - 1本の矢印が大きく影響しうる

5. デジタル補助が欲しい  
   - ホワイトボードだけでは更新漏れや見落としが起きやすい

向いている場面
==============
- 完全な総当たりはできないが、対局結果の「つながり」を重視したい大会
- 参加者数が中規模で、運営がグラフを追える大会
- 実験的イベントや、格付けの過程そのものを楽しむ大会

向いていない場面
================
- 厳密で分かりやすい最終順位が求められる大会
- 運営人数が少なく、盤外管理に手間をかけられない大会
- 参加者へ簡単な説明だけで済ませたい大会

おすすめの実務形
================
もし本当に採用を検討するなら、次の形が現実的である。  

おすすめ案:
- グラフは対局組み合わせの補助に使う
- 最終順位は勝ち星を主、グラフを補助にする
- 循環や比較不能が出たときだけグラフを参照する

つまり、
**グラフを主ルールにするより、組み合わせ誘導ルールとして使う方が安全** である。

総括
====
「格付けグラフ戦」は発想としてはかなりおもしろい。  
ただし、グラフをそのまま順位ルールにすると破綻しやすい。  

改定するなら、
- 勝敗は全記録
- 対局決定には連結成分
- 順位確定には強連結成分と層分け
- 最後は勝ち星を補助的に使う

という構造にするのがよい。  

そのうえでなお、実務では
**順位本体よりも、対局組み合わせの補助ルールとして使う方が安定**
だと思われる。

以上。
