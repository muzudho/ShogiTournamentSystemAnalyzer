## その他

```plaintext
［要求ファイル］に次のように書いてるだろ（＾～＾）

`#[Section] Step.Simulation.RuleProfileAttributes`

これって、以下みたいな `カテゴリー[インスタンス]` 表記（配列表記）を許容したら読みやすくならないかだぜ（＾～＾）？
どう思う（＾～＾）？

`#[Section] Step["Simulation"].RuleProfileAttributes`
```

```plaintext
正規表現の記述が大変になりそうだな（＾～＾）
じゃあ単純に、 `Step.A.B` を `AStep.B` にするような仕様変更だったらどうだぜ（＾～＾）？
```

```plaintext
良い案だと思うぜ（＾～＾）！
じゃあ、`Step.` 階層を `Step` を接尾辞に仕様変更する案で、まず実装計画を立ててくれだぜ（＾▽＾）！
```
