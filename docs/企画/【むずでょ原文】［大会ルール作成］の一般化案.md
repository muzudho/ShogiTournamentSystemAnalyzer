## ［タイム］の定義

```plaintext
型：　非負の整数。
スパン名：　秒、分、時間だったり、ラウンド数だったり、使い方は任意。ただのラベル文字列。

説明：
	ただのループカウンター。
	後述の TimeLoop （for文）のループカウンター。1ずつ増える。
```


## ［プレイヤー］の定義

```plaintext
データ構成：
	［名前］：　アリスさん、ボブさんなど
	［レーティング］：　Elo レーティングなど、数値で表されるプレイヤーの強さ
	［先手勝ち数］：　先手で対局に勝った回数
	［後手勝ち数］：　後手で対局に勝った回数
	［先手負け数］：　先手で対局に負けた回数。不戦敗か、棄権かはこだわらない。
	［後手負け数］：　後手で対局に負けた回数。不戦敗か、棄権かはこだわらない。
	［先手分け数］：　先手で対局が引き分けになった回数
	［後手分け数］：　後手で対局が引き分けになった回数
	［状態］：　プレイヤーの状態。例えば、対局中、待機中、棄権など。
```

## ［プレイヤーの状態］の定義

```plaintext
［ペアリング待ち］：　ペアリングされるのを待っている状態
［対局待ち］：　ペアリング後の状態を参照
［対局中］：　ペアリング後の状態を参照
［対局終了］：　ペアリング後の状態を参照
```


## ［ペア］の定義

```plaintext
データ構成：
	［先手］：　アリスさん（プレイヤー）
	［後手］：　ボブさん（プレイヤー）
	［対局開始までの残りタイム］：　１（タイム）
	［対局終了までの残りタイム］：　３（タイム）
	［対局結果］：　先手勝ち、後手勝ち、引き分け
		／／　［先手］［後手］［先手勝率］［レーティング］の情報と［乱数］があれば算出できるから、
		／／　実は［対局結果］は生成時にもう決まっている。
	進捗状態：
		［対局待ち］：　生成されてから、［対局開始までの残りタイム］が０でない間カウントダウン。
		［対局中］：　［対局開始までの残りタイム］が０になってから、［対局終了までの残りタイム］が０でない間カウントダウン。
		［対局終了］：　［対局終了までの残りタイム］が０になったとき。カウントストップ。
```

## ［対局一覧］域の定義

［初期対局予定テキスト］を変換して生成したり、対局が進行する中で動的に要素を追加したりする、対局のリスト。  

```plaintxt
データ構成：
	［ペア］のリスト
```

例：  

```plaintext
A, D
C, B

B, D

```


## ［初期対局予定ペアリング］境界の定義

［初期対局予定テキスト］を入力して、［対局一覧］を生成するための処理。  
シリアライズみたいなもの。  


## ［アクティブ対局一覧］の定義

旧名： `プレイング・プール` →　新名： `アクティブ対局一覧`  

［対局一覧］のサブリスト。現在対局中の対局のリスト。  
［対局一覧］に統合してもいいかもしれない。  

```plaintext
データ構成：
	［ペア］のリスト

メソッド：
	［空か（）］：
		アクティブ対局がないかどうかを返す。

	［更新（）］：
		リストに含むすべての ［ペア］のタイムを１進める。（１カウントダウン）
		対局の進捗状態も更新される。

	［全ての対局が終了しているか（）］：
		リストに含むすべての ［ペア］が終了しているかどうかを返す。

	［対局が終了しているペアを全て削除（）］：
		リストに含むすべての ［ペア］のうち、終了しているものを削除する。

	［ペアリング待ちの２名をペアリングしてリストに追加する（）］：
		［対局一覧］から、ペアリング待ちのプレイヤーを２名選んでペアリングして、リストに追加する。	
```

その他のメソッド：  

```plaintext
［動的ペアリング（　アクティブ対局一覧　）］：
	新しいペアをリストに追加する。
```


## ［大会結果］の定義

```plaintext
TournamentGameRecords
	※ Results という名前だと順位まで含む印象があるので Records とした。

  対局番号  |  先手プレイヤーId  |　後手プレイヤーId  |  対局開始タイム  |  対局終了タイム  |  対局結果  |  対局名  
------------+--------------------+--------------------+------------------+------------------+------------+--------------
  1         |  1 （アリス）      |  2 （ボブ）        |  1               |  2               |  先手勝ち  |  ラウンド１
```

•	TournamentMatchId または 対局番号：　一意キー
•	Stage または 段階名
•	ResultType
•	Status
* •	段階名: 予選、準決勝、決勝、本戦A組など
* •	ResultType: 先手勝ち、後手勝ち、引き分け、不戦勝、中止
* •	Status: 予定、進行中、完了、取消

> 1.	この表は大会の最終出力として使う共通結果表ですか、それとも大会進行中の内部イベントログでもありますか。

	最終出力を意識していたが、内部イベントログも兼ねると言われて気づいたぜ（＾～＾）兼ねていいんじゃないか（＾▽＾）

> 2.	対局結果は先手勝ち・後手勝ち・引き分けだけでなく不戦勝や中止も含めますか。

	対局記録はこの仕様に全部まとめたいんで、［不戦勝］［中止］など、細かくてもいいかもしれないな（＾～＾）

> 3.	対局名はラウンド1のような表示ラベルだけで十分ですか、それとも予選A組や決勝のような段階情報も別列で持ちますか。

	段階情報の方が便利かな（＾～＾）　［予選A組］、［決勝］とか（＾～＾）　プレイヤー名は別列で持っているから、あとで付けれるしな（＾～＾）

> 4.	同じ選手同士が複数回対局する大会を想定して、対局番号を大会内一意キーとして扱ってよいですか。

	それあるよな（＾～＾）　あるといいぜ（＾～＾）

```plaintext
アクティブ対局一覧 = 初期対局ペアリング( 初期対局予定テキスト )

// ループ
TimeLoop: for (t = 0; t < ∞; t += 1) {

	// ［ラウンド制］の大会は、［ペア］の［対局終了までの残りタイム］で自動的に［対局中］になるので、実は何もしなくていい。
	アクティブ対局一覧.更新()

	// 終了している［ペア］は、［終了リスト］などを作って、そこに移すなどして管理するといいかも知れない。
	// ［アリーナ式］では１組だけ対局が終了するかもしれないし、
	// ［スイス式］では、全対局が一斉に終了するかもしれない。
	アクティブ対局一覧.対局が終了しているペアを全て削除（）

	// 動的な対局追加
	//
	//		- 例えば、アリーナ制では、なるべく同じペアにならないように、 `アクティブ対局一覧.ペアリング待ちの２名をペアリングしてリストに追加する（）`
	//		- 例えば、スイス式では、勝ち星などを考慮して、対局待ちのプレイヤーをペアリングする。
	動的ペアリング（　アクティブ対局一覧　）

	// 優勝が決まった時点で大会終了するなら、ループを抜ける処理があるかも知れない。
	大会終了判定（　アクティブ対局一覧　）

	if (アクティブ対局一覧.空か())
	{
		break TimeLoop;		// ループを抜ける。
	}
}

おわり。
```
