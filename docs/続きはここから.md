## 注意点

- `RequestFileCheckWorkflow` は raw `RequestText` を返す。`ApplicationWorkflow` 側で、まず `StsaInput4RequestParser.TryParse` を試し、未対応時だけ legacy converter を呼ぶ。

- `apply_patch` が `writing outside of the project` で失敗するときは、[Codex apply_patch 回避メモ](./開発/Codex_apply_patch回避メモ_20260605.md) の PowerShell 手順を使う。

- `STSAInput/4` の `AnalysisFlowSteps=Simulation,QualityEvaluation` は形式として予約済みだが、要求ファイルからの自動実行はまだ未対応。
- 理由は、第65の新 parser が単一 `Simulation + Standard + 20局以下` だけを対象にしているため。
- 複数ステップを1ファイルで完全自動実行するには、ステップ別入力セクションの仕様追加が必要。
- smoke 実行で `Output` 配下に CSV / Markdown / `.stsa.txt` が生成・更新される。
- smoke 実行で更新された `Output` 差分は、今回の実装差分から外すため HEAD 内容へ戻してある。
- staging は触っていない。
