# ［大会ルール作成］の一般化の案

［タイム］の定義：  

```plaintext
型：　非負の整数。
スパン名：　秒、分、時間だったり、ラウンド数だったり、使い方は任意。ただのラベル文字列。

説明：
	ただのループカウンター。
	後述の TimeLoop （for文）のループカウンター。1ずつ増える。
```

［プレイヤー］の構造：  

```plaintext
データ：
	名前：　アリスさん、ボブさんなど
	レーティング：　Elo レーティングなど、数値で表されるプレイヤーの強さ
```


［ペア］の構造：  

```plaintext
データ：
	先手：　アリスさん（プレイヤー）
	後手：　ボブさん（プレイヤー）
	対局開始までの残りタイム：　１（タイム）
	対局終了までの残りタイム：　３（タイム）
	対局結果：　先手勝ち、後手勝ち、引き分け
	進捗状態：
		対局待ち：　生成されてから、［対局開始までの残りタイム］が０でない間
		対局中　：　［対局開始までの残りタイム］が０になってから、［対局終了までの残りタイム］が０でない間
		対局終了：　［対局終了までの残りタイム］が０になったとき
```


［プレイング・プール］の構造：  

```
データ：
	［ペア］のリスト

メソッド：
	空か（）
	ペアリング（）	／／　新しいペアをリストに追加する。
	更新（）		／／　リストに含むすべての ［ペア］のタイムを１進める。
	全ての対局が終了しているか（）
	対局が終了しているペアを全て削除（）
```


疑似コード：  

```plaintext

［Pairing before start］	// シミュレーションを始める前に、決まってるペアリング

［Pairing Serializer］		// ペアリングを直列にする

TimeLoop (counter by time) {	// ループ
	［プレイング・プール］

	if (プールの全ての対局が終わった、または、どこかの対局が終わった)
	{
		［Pairing at end of playing pool］	// TODO: ペアリングをリストに追加

		［Pairing Serializer］				// TODO: ペアリングを直列にする
	}
}


```
