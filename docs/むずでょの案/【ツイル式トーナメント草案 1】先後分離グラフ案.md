# 【ツイル式トーナメント草案 1】先後分離グラフ案

※ 旧題: `【大会ルール案】格付けグラフ戦_2`  
ツイル式トーナメントの中核になる、`▲/▽` を分けた比較グラフ案の整理メモ。


## 大まかに

囲碁の定先（じょうせん）のハンデを使う。  

決勝には、レーティング基準ソフト、ばらまいて参加させる。  
選手は先後やる。

対レーティング基準ソフト戦は、基準ソフトは先手に座らせる、選手の強いソフトは後手に座らせる。  
（前年の決勝ソフトに協力してもらう）

先手に誰が座るかのソフトの用意と、マッチングのルールを GPT-5.4 と相談する。  


## 細かく


記号:

- `▽` = 先手で負けた側
- `▲` = 後手で勝った側
- 矢印 `→` は「先手で負けた側から、後手で勝った側へ向ける」


（１）ホワイトボードに結果を、指向性グラフとして記録する。  
このとき、先後を分ける。  

```
（例）  
▽自分→▲いぬ  
▽ぞう→▲ねこ  
▽うさぎ→▲桂
▽ひよこ→▲いのしし
```

（２）何ラウンドか進むと、グラフができる。

まず、ラウンドごとに見ると分かりやすい。

```
（ラウンド2の例）
▽いぬ→▲ねこ
▽自分→▲ぞう
▽いのしし→▲桂
▽ひよこ→▲うさぎ
```

```
（ラウンド1と2を重ねた図）
成分A
▽自分 ──→ ▲いぬ ──→ ▲ねこ
　└────→ ▲ぞう ←── ▽ぞう

成分B
▽ひよこ ─→ ▲いのしし ─→ ▲桂
　└────→ ▲うさぎ ───→ ▲桂
```


この時点では、次の 2 つのまとまりがあるように見える。

``` 
成分A
▽自分 ──→ ▲いぬ ──→ ▲ねこ
　└────→ ▲ぞう ←── ▽ぞう

成分B
▽ひよこ ─→ ▲いのしし ─→ ▲桂
　└────→ ▲うさぎ ───→ ▲桂
```

（３）次ラウンドでは、別の成分どうしを橋渡しする対局を優先する。

``` 
（ラウンド3の例）
▽いぬ→▲うさぎ
▽いのしし→▲ぞう
```

``` 
（ラウンド3まで重ねた図）
▽自分 ──→ ▲いぬ ──────→ ▲うさぎ ─→ ▲桂
　└────→ ▲ぞう ←─────▽いのしし ←─▽ひよこ
　　　　　　　　　　　└────────→ ▲桂
```

（４）こうして、

- 小さい成分を先に作る
- 次に別成分をつなぐ対局を優先する
- 全体として大きいグラフにしていく

という流れを図で見せられる。


## 基準ソフト入りのサンプル図

基準ソフトを入れる考え方なら、こんな図もありえる。

``` 
（例）
▽自分→★基準A
▽ぞう→★基準A
▽うさぎ→★基準B
▽ひよこ→★基準B

★基準A→▲いぬ
★基準B→▲桂
```

``` 
（見た目のイメージ）
▽自分 ─┐
　　　　├→ ★基準A ─→ ▲いぬ
▽ぞう ─┘

▽うさぎ ─┐
　　　　　├→ ★基準B ─→ ▲桂
▽ひよこ ─┘
```

この図の利点は、

- 基準ソフトが「ハブ」になること
- 参加者どうしの比較だけでなく、基準点との距離感を見せられること

を説明しやすいところ。


## 図を作るときのコツ

1. まず 4人〜8人で描く
2. ラウンドごとの図を先に置く
3. そのあと重ねた完成図を置く
4. 成分A、成分Bのように名前を付ける
5. 基準ソフト入りの図は別枠にする

この順で並べると、後で読んだときにかなり追いやすい。


## 実践寄りサンプル図（シミュレーション結果ベース）

以下は、実際の品質評価結果

- `ShogiTournamentSystemAnalyzer/Output/TournamentQualityEvaluator/TournamentQualityReport/Players/Good/20260514_083354_quality_players.csv`

を参考にして、**境界付近がどう見えるか** を図にしたイメージである。  
この結果は「可変定員8ルール」を試したケースで、境界の Innov 側がかなり上へ引っ張り上げられていた。

境界付近の実データ（抜粋）:

- 銀: 期待総合順位 5.092 / 総合上位8確率 76.46%
- 桂: 期待総合順位 5.651 / 総合上位8確率 67.21%
- 香: 期待総合順位 6.285 / 総合上位8確率 55.71%
- 歩: 期待総合順位 6.989 / 総合上位8確率 42.92%
- きりん: 期待総合順位 10.000 / 総合上位8確率 57.34%
- ぞう: 期待総合順位 10.743 / 総合上位8確率 44.06%

この数字から見ると、

- Apex 下位 4 名
- Innov 上位 2 名

が、かなり近い境界帯を作っている。

``` 
（境界帯のイメージ）

上位Apex帯
▽金 ──→ ▲玉
▽銀 ──→ ▲金

境界帯
▽桂 ───→ ▲銀
▽香 ───→ ▲桂
▽歩 ───→ ▲香
▽きりん → ▲銀
▽ぞう ─→ ▲桂

下位Innov帯
▽らいおん → ▲香
▽いぬ ───→ ▲歩
```

これを図としてまとめると、例えば次のように描ける。

``` 
（境界帯を強調したASCII図）

▽金 ──→ ▲玉
  ↑
▽銀 ──→ ▲金
  ↑
▽桂 ──→ ▲銀   ←  ▽きりん
  ↑
▽香 ──→ ▲桂   ←  ▽ぞう
  ↑
▽歩 ──→ ▲香   ←  ▽らいおん
                    ↑
                 ▽いぬ
```

この図で言いたいことは、

- Apex 下位だけで階段ができるのではなく
- Innov 上位も同じ階段へ横から食い込んでくる

という点である。

つまり、可変定員8ルールを採ると、
**境界のグラフはきれいになるが、Apex上位帯の固定性は崩れやすい**、というシミュレーション結果を視覚的に表現できる。


## さらに実践的にするなら

もし次に図をもっと実戦寄りにするなら、次の 2 種類を描き分けるとよい。

1. **改善案A単独の図**
   - Apex 上位帯が比較的きれいに守られる図

2. **可変定員8ルールの図**
   - Innov 上位が境界へ食い込んでくる図

この 2 枚を並べると、
「公平性は上がるが、切れ味は落ちる」
という違いがかなり直感的に伝わる。
