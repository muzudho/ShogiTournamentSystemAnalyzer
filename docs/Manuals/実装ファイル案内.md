## まず全体像

アプリ本体は `ShogiTournamentSystemAnalyzer` フォルダーにあります。

- `Program.cs`
  - エントリーポイントです
  - 起動メッセージ、入力ソース設定、モード分岐だけを持ちます
  - 「最初にどこから始まるか」を追いたいときの入口です
- `Presentation`
  - コンソール表示や入力の責務です
- `Application`
  - モード進行、入力適用、出力先決定、検証などの**アプリ進行役**です
- `Infrastructure`
  - CSV 出力や入力パースなど、外側とのやりとりです
- `Domain`
  - ルール、計算、モデル、列挙型などの**中核ロジック**です
- `Examples`
  - 人が読むためのサンプル CSV とメモです
- `Inputs`
  - `--input-file` で流す実行用入力ファイルです
- `Output`
  - 通常実行時の出力先です

## 起点になるファイル

### `ShogiTournamentSystemAnalyzer/Program.cs`
最初に読むならここです。

- `ConfigureInputSource(args)`
  - `--input-file` を使う流れへ入ります
- `ReadMode()`
  - モード選択へ入ります
- `RunStandardMode()`
  - 通常モードへ入ります
- `RunFinalStageMode()`
  - 本戦専用モードへ入ります
- `RunQualityEvaluationMode()`
  - 品質評価モードへ入ります

「どのファイルがどこから呼ばれているか」を大づかみしたいときは、まずここから追うのが早いです。

## 最初に読む順

初見で追うなら、次の順がおすすめです。

1. `ShogiTournamentSystemAnalyzer/Program.cs`
   - 起動から各モードへどう分岐するかを見る
2. `Application/Modes/*Mode.cs`
   - そのモードの骨格を見る
   - `StandardMode.cs` / `FinalStageMode.cs` / `QualityEvaluationMode.cs`
3. `Application/Modes/*Input.cs` / `*Execution.cs` / `*Output.cs`
   - 入力、実行、出力のどこを直すかで追う
4. `Application/Validation` と `Application/Reporting`
   - 検証やレポート行生成を見たいときに読む
5. `Infrastructure/Parsing` と `Infrastructure/Csv`
   - 入出力フォーマットを直したいときに読む
6. `Domain/Calculation` / `Domain/Rules` / `Domain/Enums` / `Domain/Models`
   - 計算の意味やルールの本体を直したいときに読む

「まず骨格、そのあと入力・実行・出力、最後に Domain」という順で見ると迷いにくいです。

## フォルダー別の役割

### `Presentation/Console`
コンソール入出力まわりです。

- `ConsolePrompts.cs`
  - 案内文、選択肢、プロンプト表示を直したいとき
- `ConsoleRuleSelections.cs`
  - ルール選択 UI を直したいとき
- `ConsoleInputReaders.cs`
  - コンソールからの読み取り手順を直したいとき
- `ConsoleSamples.cs`
  - サンプル表示や入力例を直したいとき
- `ConsoleResultPrinter.cs`
  - コンソールに出る結果表、品質評価サマリー表示を直したいとき

### `Application/Modes`
各モードの進行役です。現在の入口は、まず「対局シミュレーション / 品質評価」を選び、その後に「通常ルール / 本戦ルール」を選ぶ形です。

- `SimulationStandardMode.cs`
	- 対局シミュレーション / 通常ルールの進行の骨格を直したいとき
- `SimulationStandardModeInput.cs`
	- 対局シミュレーション / 通常ルールの入力読取を直したいとき
- `SimulationStandardModeExecution.cs`
	- 対局シミュレーション / 通常ルールの計算実行、結果表示、CSV 出力を直したいとき
- `SimulationFinalStageMode.cs`
	- 対局シミュレーション / 本戦ルールの進行の骨格を直したいとき
- `SimulationFinalStageModeInput.cs`
	- 対局シミュレーション / 本戦ルールの入力読取や検証を直したいとき
- `SimulationFinalStageModeExecution.cs`
	- 対局シミュレーション / 本戦ルールの計算分岐、文脈表示、出力書き出しを直したいとき
- `QualityEvaluationMode.cs`
	- 品質評価 / 通常ルール・本戦ルールの進行の骨格や入力解釈を直したいとき
- `QualityEvaluationExecution.cs`
  - 品質評価の実行手順、スイープ、試行設定を直したいとき
- `QualityEvaluationOutput.cs`
  - 品質評価の表示、出力先選択、CSV 出力まわりを直したいとき

「モードの流れを変えたい」ときは、まずここを見ます。

### `Application/Validation`
入力内容や大会条件の整合性チェックです。

- `FinalStageValidators.cs`
  - 本戦 16 人条件、Apex 重複、グループ整合などの検証を直したいとき

### `Application/Reporting`
表示・レポート用の行データや集計を作る場所です。

- `ResultBuilders.cs`
  - 結果表の列、品質評価サマリー、相関係数計算を直したいとき

### `Application/Paths`
レポートの出力先やファイル名を決める場所です。

- `ReportOutputPathBuilder.cs`
  - `docs/Reports/Good` / `Bad` への振り分けや既定ファイル名を直したいとき

### `Application/Input`
入力ファイルの適用です。

- `InputSourceConfiguration.cs`
  - `--input-file` の解釈、コメント行除去、`#[Enter]` などの入力前処理を直したいとき

### `Application/Execution`
実行時間や共通の実行制御です。

- `SimulationBudget.cs`
  - シミュレーション時間制限や確率正規化を直したいとき

### `Application/Helpers`
複数モードで共有する軽い進行補助です。

- `ModeSupportHelpers.cs`
  - モード共通の補助処理を直したいとき

### `Infrastructure/Csv`
CSV の出力処理です。

- `CsvOutputHelpers.cs`
  - CSV 文字列の整形や補助を直したいとき
- `ResultCsvWriter.cs`
  - 結果 CSV、品質評価 CSV、参考対局 CSV の書き出しを直したいとき

### `Infrastructure/Parsing`
CSV や対局表入力の読み取りです。

- `InputParsers.cs`
  - 選手一覧 CSV、対局 CSV、Round/Black-White/対局記号表の読み取りを直したいとき

### `Domain/Calculation`
大会結果を計算する中核です。

- `StandardCalculationEngine.cs`
  - 通常モードの厳密計算やシミュレーションを直したいとき
- `FinalStageCalculationEngine.cs`
  - 本戦専用モードの順位計算、境界救済、追加 Apex、可変 Top8 を直したいとき

### `Domain/Math`
数式・レーティング・書式の補助です。

- `RatingMath.cs`
  - 先手勝率とレーティング換算、表示フォーマットを直したいとき

### `Domain/Models`
基本データ構造です。

- `TournamentModels.cs`
  - `Participant`、`Match`、`CalculationResult` など基本モデルを直したいとき
- `QualityEvaluationModels.cs`
  - 品質評価用の行モデルやサマリー構造を直したいとき

### `Domain/Enums`
全体で使う列挙型です。

- `TournamentEnums.cs`
	- `TournamentRuleSetMode`、本戦グループ、境界救済モードなどの列挙値を直したいとき

### `Domain/QualityEvaluation`
品質評価専用の入力・設定モデルです。

- `QualityEvaluationInput.cs`
  - 品質評価の入力まとまりを直したいとき
- `QualityEvaluationExecutionOptions.cs`
  - 実行回数などの設定構造を直したいとき
- `QualityEvaluationOutputOptions.cs`
  - 品質評価の出力設定を直したいとき
- `QualityEvaluationRuleDefinition.cs`
  - 品質評価で使うルール定義を直したいとき

### `Domain/Rules`
ルールの定義、ラベル、純粋ロジックです。

- `TournamentRuleSetRule.cs`
	- `Neutral` / `Twill` のラベルを直したいとき
- `AdditionalApexPlacementRule.cs`
  - 追加 Apex の扱いを直したいとき
- `BoundaryRescueRule.cs`
  - 境界救済の選択や意味を直したいとき
- `FinalStageGroupingRule.cs`
  - 本戦グループ分けの扱いを直したいとき
- `TwillTournamentRule.cs`
  - ツイル式ルールの処理を直したいとき
- `VariableTop8Rule.cs`
  - 可変 Top8 の扱いを直したいとき

## 変更時のおすすめ確認順

コードを直したあと、次の順で確認すると安全です。

1. 変更した責務の近いファイルを見直す
   - 例: `Application/Modes` を直したら、対応する `Input` / `Execution` / `Output` も軽く確認する
2. ビルドする
   - まず `dotnet build` でコンパイルが通るか確認する
3. 必要なら入力ファイルで軽く動かす
   - `Inputs/Smoke` は軽い確認向きです
   - `Inputs/Bench` は単発評価向きです
   - `Inputs/Sweeps` は条件を振って確認したいとき向きです
4. 出力先や CSV を確認する
   - `Output`
   - `docs/Reports`
5. 説明書に影響があれば docs も更新する
   - `README.md`
   - `docs/README.md`
   - `docs/Manuals/README.md`
   - `docs/Manuals/実装ファイル案内.md`

## 実行用ファイルと資料の置き場

### `ShogiTournamentSystemAnalyzer/Examples`
人が読むための見本やメモです。

- 詳細: `ShogiTournamentSystemAnalyzer/Examples/README.md`

- 対局記号表や入力形式の見本を探したいとき
- メモ付きで理解したいとき

### `ShogiTournamentSystemAnalyzer/Data/Inputs`
自動実行用の入力ファイルです。

- `Smoke`
  - 軽い動作確認
- `Bench`
  - 単発の評価や比較
- `Sweeps`
  - 条件を振って繰り返し回す入力

### `ShogiTournamentSystemAnalyzer/Data/Players`
再利用する選手データです。

### `ShogiTournamentSystemAnalyzer/Data/Matches`
再利用する対局表や参考対局表です。

### `ShogiTournamentSystemAnalyzer/Data/FinalStage`
本戦専用モードのグループ対応表や補助 CSV です。
再利用する選手データの置き場です。

### `ShogiTournamentSystemAnalyzer/Data/RuleSets`
保存して再利用する大会ルールの置き場です。

### `docs/Reports`
大会品質評価レポートや比較結果の保存先です。

- `Good`
  - 良い結果として分けたいレポート
- `Bad`
  - 悪い結果として分けたいレポート

## 迷ったときの見方

1. まず `Program.cs` で入口を見る
2. 次に `Application/Modes` でその機能の流れを見る
3. 入出力の話なら `Presentation` / `Infrastructure` を見る
4. 計算やルールの話なら `Domain` を見る
5. サンプルや実行例は `Examples` / `Data` / `docs/Reports` を見る
