# 品質評価

## 概要
品質評価モードは、本戦ルールが参加者の実力をどの程度そのまま順位へ反映できているかを確認するためのモードです。

単発の品質評価だけでなく、先手勝率を一定範囲で変化させる **n% スイープ実験** にも対応しています。

- モード番号: `3`
- 主な用途:
  - 本戦ルール案の比較
  - `Neutral` と `Twill` の比較
  - `Apex / Innov` 分割あり・なしの比較
  - 改善案の `On / Off` 比較

## 基本的な考え方
品質評価では、まず **ニュートラル** を基準にします。
ニュートラルより良い大会ルールを `Good`、悪くなるルールを `Bad` として扱います。

`Apex / Innov` を `Off` にした場合は、品質評価でも `Neutral / Twill` を切り替えられます。

`Twill` の評価では、`Neutral` を基準にして次の指標がどう変わるかを見る使い方を想定しています。

- Spearman 相関
- 平均順位ずれ
- 上位8残留
- Elo1位の総合1位確率

## 評価指標
品質評価モードでは、順位分布そのものに加えて、次の品質指標を表示し、CSV 出力も行います。

- Spearman 相関
- 平均順位ずれ
- Elo上位8名の総合上位8位残留人数（平均）
- Elo1位の総合1位確率
- 最大不利益 / 最大利益

### 指標の見方
- **Spearman 相関**
  - Elo順位と最終順位の並びがどれだけ近いかを見る指標です。
  - 値が高いほど、実力順を保ちやすいと見なせます。
- **平均順位ずれ**
  - 各参加者について、Elo順位と期待総合順位の差の絶対値を平均したものです。
  - 小さいほど、順位のぶれが少ないと見なせます。
- **上位8残留**
  - Elo 上位 8 名が、総合上位 8 位にどれだけ残るかの平均です。
  - 本戦進出者選抜の妥当性を見たいときに使います。
- **Elo1位の総合1位確率**
  - 最上位実力者が優勝できる確率です。
  - 最強者を正しく上に置けるかの目安になります。
- **最大不利益 / 最大利益**
  - どの参加者が最も順位面で損・得をしているかを見るための指標です。

## 入力
品質評価モードの基本入力は、本戦専用モードと同じです。

1. 同Elo対局時の先手勝率 (%)
2. 選手一覧CSV
3. Apex / Innov の分け方（On / Off）
4. `Off` のときだけ順位ルールセット（`Neutral / Twill`）
5. `On` のときだけグループ対応CSV
6. `On` のときだけ本戦不出場Apex一覧CSV（省略可）
7. `On` のときだけ本戦不出場Apexの扱い（Off / On）
8. `On` のときだけ境界救済戦の有無（Off / On）
9. `On` のときだけ可変定員8ルールの有無（Off / On）
10. 対局CSV または Round/Black-White/対局記号表
11. 参考対局CSV または Round/Black-White/対局記号表（省略可）
12. 単発評価または n% スイープ実験の選択
13. 必要に応じてシミュレーション試行回数
14. レポート出力先パスまたはフォルダーパス

## 切り替え可能なルール
品質評価モードでは、実験用に次を切り替えられます。

### Apex / Innov の分け方
- `Off`: フラットに順位づけする
- `On`: Apex / Innov を使う

### 本戦不出場Apexの扱い
- `Off`: Innov より前に順位帯を確保する（現行案）
- `On`: 総合順位へ挿入しない（改善案A）

### 境界救済戦
- `Off`: 使わない
- `On`: Apex最下位相当と Innov 最上位相当で救済戦を行う

### 可変定員8ルール
- `Off`: 定員8固定
- `On`: 本戦不出場Apex人数ぶんだけ Innov 上位を総合上位8へ引き上げる

## n% スイープ実験
品質評価モードでは、単発評価のほかに **n% スイープ実験** を選べます。
これは先手勝率を一定範囲で振りながら、各 n% に対する品質指標の変化をまとめて CSV 出力する機能です。

例えば、50% から 100% まで 5 刻みで評価すると、先手有利が強くなるほどルールの公平性や実力反映性がどう変わるかを比較できます。

実行用入力ファイルは主に次に配置します。

- `ShogiTournamentPairingAnalyzer/Inputs/Smoke`
- `ShogiTournamentPairingAnalyzer/Inputs/Bench`
- `ShogiTournamentPairingAnalyzer/Inputs/Sweeps`

## 参考対局
本戦専用モード / 品質評価モードでは、**参考対局** を別入力で受け付けられます。

- 参考対局は表示・CSV保存されます
- **大会記録や品質評価には含めません**

大会本体と、補足として見せたい参考結果を分けたいときに使います。

## レポート運用
品質評価レポートは、必要に応じて次のように分けて管理できます。

- `docs/Reports/Good`
- `docs/Reports/Bad`

基準は **ニュートラル** です。
ニュートラルより良いものを `Good`、悪いものを `Bad` とし、必要なら **評価メモ** を 1 行残して運用します。
