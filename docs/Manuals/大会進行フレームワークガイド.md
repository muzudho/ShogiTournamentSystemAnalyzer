# 大会進行フレームワークガイド

## これは何か
大会進行フレームワークは、固定の総当たり戦だけでなく、より一般化した大会記録をそのまま入力して実行するためのモードです。

通常ルールが `選手一覧CSV + 対局CSV` を中心に扱うのに対し、このモードでは次を分けて持ちます。

- `PlayersCsvPath`
- `StagesCsvPath`
- `TournamentMatchRecordsCsvPath`
- `RuleFilePath`（大会ルール DSL、任意）

## どこで選ぶか
起動後に次を選びます。

1. `対局シミュレーション`
2. `大会進行フレームワーク`

## 入力順
このモードでは、主に次を順に入力します。

1. 同Elo対局時の先手勝率 (%)
2. 順位ルールセット（`Neutral / Twill / Twill+CommonOpp`）
3. 選手一覧CSVのパス
4. ステージ一覧CSVのパス
5. 大会対局記録CSVのパス
6. 大会ルールDSLファイルのパス（省略可）
7. 乱数シード（省略可）
8. 試行回数（省略可）
9. aggregate 結果CSVの出力先（省略可）

## 主な入力ファイル
### Players CSV
選手ID、名前、レーティングを持つ一覧です。

### Stages CSV
ステージID、ステージ名、種別、親子関係、表示順を持つ一覧です。

### TournamentMatchRecords CSV
対局番号、ステージ、先手、後手、開始タイム、終了タイム、状態、結果、ラウンド番号などを持つ大会基底表です。

### RuleFilePath
大会ルール DSL のファイルです。省略できます。

## 出力の見方
このモードでは、出力が 3 系統に分かれます。

### 1. aggregate 順位表
- `tournament_framework_aggregate_result_*.csv`
- `tournament_framework_aggregate_result_*.md`
- 複数回試行を集計した順位表です
- 優勝確率や平均順位を見るときはこれを見ます

### 2. representative 順位表
- `tournament_framework_representative_ranking_*.csv`
- `tournament_framework_representative_ranking_*.md`
- 代表実行 1 件の順位表です
- コンソールの `代表実行順位` と同系統です

### 3. representative 大会最終状態
- `tournament_match_records_representative_*.csv`
- `tournament_match_records_representative_*.md`
- 代表実行 1 件の対局記録です
- aggregate 順位表とは 1 対 1 には対応しません

## Markdown の流れ
- aggregate 順位表 Markdown から representative 順位表 Markdown へ移れます
- representative 順位表 Markdown から representative 大会最終状態 Markdown へ移れます
- representative 大会最終状態 Markdown から aggregate 順位表 Markdown と representative 順位表 Markdown へ戻れます

## サンプル
