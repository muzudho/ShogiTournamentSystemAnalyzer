# 入力ファイル仕様

## 概要
このアプリは `--input-file` を使って、対話入力の代わりにテキストファイルから入力できます。

値だけを並べると意味が分かりにくくなりやすいため、現在の入力ファイルでは `#[Prompt]` と `#[Enter]` を使う運用を想定しています。

次世代方式として、`#[Format] STSAInput/2` と `#[Section] ... #[EndSection]` を使うセクション型プロトコルも検討しています。

- 仕様ドラフト: [STSAInput/2 仕様](./STSAInput2仕様.md)

## 置き場所
実行用入力ファイルは、`Data/Inputs` 配下に置く運用を基本にします。

- `ShogiTournamentSystemAnalyzer/Data/Inputs/Smoke`
  - 1 回や少数回のスモークテスト用
- `ShogiTournamentSystemAnalyzer/Data/Inputs/Bench`
  - 単発評価や軽いベンチマーク用
- `ShogiTournamentSystemAnalyzer/Data/Inputs/Sweeps`
  - n% スイープ実験用

関連フォルダー:

- `ShogiTournamentSystemAnalyzer/Examples`
	- 人が読むための見本、対局記号表、メモ
	- 詳細: `ShogiTournamentSystemAnalyzer/Examples/README.md`
- `ShogiTournamentSystemAnalyzer/Data/Players`
  - 再利用する選手データの置き場
- `ShogiTournamentSystemAnalyzer/Data/Matches`
  - 再利用する対局表、参考対局表の置き場
- `ShogiTournamentSystemAnalyzer/Data/FinalStage`
  - 本戦専用モードのグループ対応表や補助CSVの置き場
- `ShogiTournamentSystemAnalyzer/Data/RuleSets`
  - 保存して再利用する大会ルールの置き場
- `ShogiTournamentSystemAnalyzer/Output`
  - 実行で自動生成される CSV / Markdown の保存先
  - `Ranking/FinalRanking`: 通常結果、本戦結果、aggregate 結果
  - `Simulation/TournamentFinalState`: 大会最終状態、代表実行の対局記録
  - `TournamentQualityEvaluator/TournamentQualityReport`: 品質評価の Summary / Players / Sweeps

## 実行方法
```powershell
dotnet run --project .\ShogiTournamentSystemAnalyzer\ShogiTournamentSystemAnalyzer.csproj -- --input-file .\ShogiTournamentSystemAnalyzer\Data\Inputs\Bench\quality_input_[黒8x白8].txt
```

## コメントと終端記法
入力ファイルでは `#` で始まる行をコメントとして無視できます。

- `#[Prompt]`
  - その直後の値が何を表すかを書くための注記です
- `#[Enter]`
  - 空行 1 行として扱われます

この仕組みにより、空行の意味が分かりにくい問題を避けつつ、対話入力の流れをそのままファイルに写せます。

### 例
```plaintext
#[Prompt] モード番号を入力してください [1]
3
#[Prompt] 同Elo対局時の先手勝率(%)を入力してください [51]
51

#[Prompt] 選手 / Player 一覧CSVを貼り付けてください。入力終了は #[Enter] です。
name,elo
Alice,1500
Bob,1480
#[Enter]
```

## 基本ルール
- `#` で始まる行は無視されます
- `#[Enter]` は空行として扱われます

## 今後の入力プロトコル案

現行方式は、対話画面の流れをそのまま入力ファイルへ写しやすい反面、プロンプト文言や入力順の変更に影響されやすい面があります。

そのため、将来的には UI 文言から独立した安定キー中心の方式として、`STSAInput/2` を検討しています。

- `#[Format] STSAInput/2`
- `#[Section] Meta`
- `FirstPlayerWinRatePercent=51`
- `#[Section] PlayersCsv`
- `#[Section] MatchesInput`
- `#[EndSection]`

詳細は [STSAInput/2 仕様](./STSAInput2仕様.md) を参照してください。
- 実際の空行も従来どおり入力終了に使えます
- 入力順はモードごとに異なります
- 詳細な入力順は [モード別ガイド](./モード別ガイド.md) を参照してください
- 入力値の誤りが続くときは、主要な入力は 10 回で中断します

## 入力失敗時の扱い
- 選択入力、数値入力、CSV入力などの主要な入力は、誤りが 10 回続くと中断します
- 中断時は、最後に失敗した入力内容の種別とエラー理由を表示します
- 入力ファイル実行でも同じ扱いです

## 向いている使い方
- 再現可能な実験の保存
- ベンチマークやスモークテストの定型化
- 品質評価の実行手順共有
- 数値入力に説明を添えた保守しやすい実験ファイルの作成

## STSAInput/2 の空ルールサンプル
空ルールでは、`RuleProfileMode=Empty` を指定します。

```plaintext
#[Format] STSAInput/2

#[Prompt] EmptyRuleMinimal
#[Section] Meta
AnalysisFlowMode=Simulation
RuleProfileMode=Empty
OutputPath=ShogiTournamentSystemAnalyzer\Output\Ranking\FinalRanking\empty_rule_result_[最小].csv
#[EndSection]

#[Section] Output
OutputPath=ShogiTournamentSystemAnalyzer\Output\Ranking\FinalRanking\empty_rule_result_[最小].csv
#[EndSection]
```

品質評価の既定出力先は、用途ごとに次へ分かれます。

- `ShogiTournamentSystemAnalyzer/Output/TournamentQualityEvaluator/TournamentQualityReport/Summary`
- `ShogiTournamentSystemAnalyzer/Output/TournamentQualityEvaluator/TournamentQualityReport/Players`
- `ShogiTournamentSystemAnalyzer/Output/TournamentQualityEvaluator/TournamentQualityReport/Sweeps`
