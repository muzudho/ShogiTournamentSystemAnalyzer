# プロジェクトの考え方

この文書は、README から切り出した背景説明をまとめたものです。  
README では最初の利用導線を優先し、ここでは用語、仮定、狙いを整理します。

## 大会ルール（Tournament Rule）とゲームのルール（Game Rule）の違い

例えば、［駒落ち］は将棋のルールであって、囲碁のルールではないことから、（固有の）［**ゲームのルール**］に分類されます。  
例えば、Elo レーティングは、２人用の対戦ゲームであれば、どのゲームの大会ルールにも使えることから、（一般的な２人用対戦ゲームの）［**大会ルール**］の部品に分類されます。  

> [!CAUTION]
> このプロジェクトは、［ゲームのルール］を検討するためのものではなく、［大会のルール］を検討するためのものです。

別の言い方をすると、  

> [!IMPORTANT]
> このプロジェクトは、［ゲームのルール］を変えません。

## 選手（Player）と参加者（Participant）の違い

たとえばコンピューター将棋大会では、実際に対局する「選手」はソフトであり、人間は「参加者」です。  

こうした事情を承知した上で、このプロジェクトでは、コンピューターソフトが対局主体になる大会だけでなく、人間が対局主体になる大会にも使えるようにすることを考えています。  

そこで、このプロジェクトでは大会システム分析用の抽象化として、順位計算や対局表の対象になる主体を、基本的に **選手（Player）** という言葉を使っています。  

## むずでょの仮定： `［大会ルール］だけで、［先手勝率問題］を解決できる`

このプロジェクトは、`［大会ルール］だけで、［先手勝率問題］を解決できる` という仮定のもとに進めています。  
仮定ですから、結果は以下の３つです。

- 成立する
- 成立しない
- まだ決着しない

どの結果になるかを確かめるのが、このプロジェクトの１番の目的です。  

## 実験の進め方

このプロジェクトは、将棋大会の［大会ルールの部品］を試作し、シミュレーションで品質を比較しながら、より良い［大会ルール］を作っていくための実験場です。  

- ルール案を思いつく
- ルールの部品として組み込む
- 回して比べる
- もっと良くする

この繰り返しで、大会ルールを育てていきます。

## GPT-5.4 を前提にした運用

> [!TIP]
> このプロジェクトでは、  
> （１）このプロジェクト自体のコーディング  
> （２）このプロジェクトで作ったツールで大会ルールのシミュレーション検証  
> （３）新しい大会ルールの追加  
> のいずれも、GPT-5.4 にやってもらうことを想定しています。  

このプロジェクトは、GPT-5.4 に与える思考のフレームワークと考えてください。  
わたし（人間；むずでょ）は、思考のフレームワークがあらぬ方向に行かぬように整えたり、つまづいていれば小石を取り除いたり、まるで庭の草むしりだけするかのようなことをしています。
