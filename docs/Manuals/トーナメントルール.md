# トーナメントルール

## 前提ルール
- 各対局は黒番・白番を持つ
- 同 Elo 対局時の黒番勝率を 0〜100% で指定する
- 勝率は Elo 差と黒番補正から計算する
- 同点順位は該当順位帯に等分配する

## 順位ルールセット
このアプリでは、順位づけのルールセットとして `Neutral`、`Twill`、`Twill+CommonOpp` を扱えます。

## Neutral
- 既定の順位ルールです
- 各シナリオでの勝ち数をもとに順位を決めます
- 同点順位は該当順位帯に等分配します

### 向いている用途
- 基準ルールとしての比較
- 単純な勝ち数順位の確認
- 品質評価でのベースライン

## Twill
- **ツイル式トーナメント** の順位ルールです
- 各対局結果を `▲ / ▽` の比較グラフとして読み、最後に **重箱表** 相当の指標へたたんで順位を決めます

### 優先順
概ね次の優先順で比較します。

1. 合計右ノード数
2. 合計勝ち星
3. 合計最長経路長
4. `▽` 側で負けた相手の強さ
5. 直接対戦

### 向いている用途
- 単純な勝ち数だけでは見えにくい対戦構造の反映
- 順位づけ規則の比較実験
- `Neutral` との差分観察

## Twill+CommonOpp
- `Twill` を土台に、同順位グループの細分化で **共通相手比較** を追加した派生ルールです
- 2人の選手が共通して対戦した相手だけを取り出し、結果差が出たときに比較材料として使います
- 共通相手が少ない比較は不安定なので、1人だけなら弱く、2人ならやや弱く、3人でかなり採用、4人以上でフル採用、という **信頼度** を掛けて扱います

### 優先順
概ね次の優先順で比較します。

1. 合計右ノード数
2. 合計勝ち星
3. 合計最長経路長
4. `▽` 側で負けた相手の強さ
5. 共通相手比較（信頼度付き）
6. 直接対戦

### 向いている用途
- `Twill` の考え方を保ったまま、同順位グループをもう一段だけ自然に崩したいとき
- 直接対戦だけに頼らず、対戦表全体の中で似た相手への勝敗差も見たいとき
- 品質評価で `Twill` 本体と改良案を並べて比べたいとき

## どのモードで選べるか
- 通常モードでは常に `Neutral / Twill / Twill+CommonOpp` を選べます
- 本戦専用モードと品質評価モードでは、`Apex / Innov` の分け方を `Off` にしたときだけ `Neutral / Twill / Twill+CommonOpp` を選べます
