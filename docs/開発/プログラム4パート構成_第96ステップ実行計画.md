# プログラム4パート構成 第96ステップ実行計画

## 目的

第95ステップで `SimulationDomain` が `SimulationMainlineResult` を受け取れるようになった。

今回は、`SimulationRequestDispatcher` からも戻り値付き入口を呼べるようにし、`SimulationDomain` の外側へは mainline 専用型ではなくシミュレーション域の結果型を返す。

## 今回の小さな先行作業

- `SimulationDomainResult` を追加し、`SimulationResult` と `FinalRankingResult` を包む。
- `SimulationDomain.TryExecute(AnalysisStepRequest step, out SimulationDomainResult? result)` を本命入口にする。
- 既存の `SimulationDomain.TryExecute(AnalysisStepRequest step)` は互換 wrapper として残す。
- `SimulationRequestDispatcher.TryExecute(AnalysisStepRequest step, out SimulationDomainResult? result)` を追加する。
- 既存の `AnalysisRequestDispatcher` の実行順と戻り値なし実行は変えない。

## 次の実装候補

1. `AnalysisRequestDispatcher` の内部でシミュレーション域の結果を受け取り、次ステップへ渡す置き場を作る。
2. `FinalRankingDomain` に、順位行生成または出力の入口をさらに寄せる。
3. `StandardSimulationMainline` / `FinalStageSimulationMainline` から最終順位表出力を外へ出す。
4. `AnalysisRequestDispatcher` を `AnalysisDomain` 風の直列実行へ寄せる。
