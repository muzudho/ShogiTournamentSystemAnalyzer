# プログラム4パート構成 第24ステップ実行計画

## 目的

`ApplicationRun` にある起動準備処理を分離し、アプリケーション実行本体の見通しを良くする。

アプリケーションの時間予算開始と導入表示は、分析実行そのものではなく起動時の準備として `ApplicationStartup` にまとめる。

## 対象

- `ShogiTournamentSystemAnalyzer/Application/Execution/ApplicationRun.cs`
- `ShogiTournamentSystemAnalyzer/Application/Execution/ApplicationStartup.cs`

## 作業

1. `ApplicationStartup` を追加する。
2. `SimulationTimeBudget.BeginApplicationBudget()` と `ProgramConsoleGuide.PrintProgramIntroduction()` を `ApplicationStartup.Start()` に移す。
3. `ApplicationRun.Run(args)` は `ApplicationStartup.Start()`、入力元設定、分析実行、入力セッション完了の順にする。
4. 外部挙動は変えない。

## 確認

- `dotnet build .\ShogiTournamentSystemAnalyzer\ShogiTournamentSystemAnalyzer.csproj`
- 空ルールの要求ファイル入力スモーク
- 手動入力保存のスモーク

## 今回は触らないこと

- `Program.cs` の例外処理
- `Analysis` の内部処理
- 入力セッションの内部構造
- 要求ファイルの入出力書式
