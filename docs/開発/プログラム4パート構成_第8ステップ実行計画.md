# プログラム4パート構成 第8ステップ実行計画

## 目的

第7ステップで STSA の値 parse を `StsaInputValueParser` に分離した。
第8ステップでは、既存対話入力列を作るための小さな追記ヘルパーを `LegacyInputLineBuilder` に分離する。

## 現状

`RequestInputFileReader` の末尾には、次の2つの helper が残っている。

- `AppendDelimitedSection`
- `AppendEndTerminatedSection`

どちらも STSA の構文解析や値解釈ではなく、既存対話入力列へ section 本文を並べるための補助である。

## 今回やること

1. `Application/Input/LegacyInputLineBuilder.cs` を追加する。
2. `AppendDelimitedSection` と `AppendEndTerminatedSection` を移す。
3. `RequestInputFileReader` は `using static` で使う。
4. 生成される legacy 入力列は変えない。

## 今回やらないこと

- legacy 入力列そのものの仕様変更
- STSAInput/2 と STSAInput/3 の converter 分割
- 変換結果のモデル化

## 完了条件

- legacy 入力列 helper が `LegacyInputLineBuilder` に分離されている。
- `dotnet build` が成功する。
- `--input-file` 経由の空ルール smoke 実行が成功する。
