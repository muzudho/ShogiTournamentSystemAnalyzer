# プログラム4パート構成 第68ステップ実行計画

## 目的

`STSAInput/4` の `Simulation + FinalStage` を、legacy 入力列へ変換せず `AnalysisRequest` 直通ルートで実行する。

## 方針

- `FinalStageSimulationRequest` を追加する。
- `StsaInput4RequestParser` で `PlayersCsv` / `GroupMapCsv` / `AdditionalApexPlayersCsv` / `MatchesInput` / `ReferenceMatchesInput` を直接読む。
- 本戦シミュレーション用の `FinalStageModeSimulationContext` を `AnalysisRequestDispatcher` で組み立てる。
- `FinalStageSimulationMainline` に、要求ファイルの出力先を使うための `Run(..., outputPathOverride)` を追加する。
- 20局超の本戦シミュレーションは、現時点では従来どおり legacy fallback に残す。

## 実装内容

- `AnalysisRequest.cs`
  - `FinalStageSimulationRequest` を追加。
- `AnalysisRequestDispatcher.cs`
  - 本戦シミュレーション要求を `FinalStageSimulationMainline.Run(...)` へ渡す分岐を追加。
- `StsaInput4RequestParser.cs`
  - `Simulation + FinalStage + 20局以下` 分岐を追加。
  - 本戦グループ、追加 Apex、境界救済、参考対局、出力先の直接パースを追加。
- `FinalStageSimulationMainline.cs`
  - 標準シミュレーションと同じ出力先 override を追加。

## 検証

- `dotnet build .\ShogiTournamentSystemAnalyzer\ShogiTournamentSystemAnalyzer.csproj`
- `dotnet run --project .\ShogiTournamentSystemAnalyzer\ShogiTournamentSystemAnalyzer.csproj -- --input-file ".\Inputs\Smoke\simulation_input_[本戦16]_[FinalStage_Exact]_[STSAInput4_smoke].request.txt"`
- `git diff --check`
