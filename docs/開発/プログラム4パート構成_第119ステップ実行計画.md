# プログラム4パート構成 第119ステップ実行計画

## 目的

`STSAInput/5` の `RuleProfileAttributes` 入力が、標準品質評価だけでなく、本戦品質評価と `Simulation,QualityEvaluation` 複数ステップでも実行できることを smoke で固定する。

第118ステップでは `STSAInput/5` の入口と標準品質評価 smoke を追加した。今回は parser の仕様を変えず、既存 `STSAInput/4` smoke と同等の入力を属性形式へ写して、互換ラベルなしの要求ファイルで実行ルートを確認する。

## 実施内容

1. 本戦品質評価の `STSAInput/5` smoke を追加する。
   - `RuleProfileMode=FinalStage` は書かない。
   - 単一ステップ用の `RuleProfileAttributes` セクションを使う。
   - 本戦向け属性は `UsesFinalStageGrouping=On`、`UsesAdditionalApexPlacement=On`、`UsesBoundaryRescue=On`、`UsesVariableTop8=On` とする。
2. 標準 `Simulation,QualityEvaluation` 複数ステップの `STSAInput/5` smoke を追加する。
   - `Step.Simulation.RuleProfileAttributes` と `Step.QualityEvaluation.RuleProfileAttributes` を使う。
   - どちらの step にも `RuleProfileMode` は書かない。
3. 既存の `STSAInput/5` 標準品質評価 smoke も再実行し、既存ルートを壊していないことを確認する。

## 変えないこと

- `StsaInput4RequestParser` の正規化処理。
- Writer の既定出力形式。依頼ログは引き続き `STSAInput/4`。
- 品質評価・シミュレーションの計算ロジック。
- 既存 `STSAInput/4` smoke。

## 実施結果

- `Inputs/Smoke/quality_input_[トップ集団大きめ]_[FinalStage_Single10]_[STSAInput5_smoke].request.txt` を追加した。
- `Inputs/Smoke/analysis_input_[先手8x後手8]_[Simulation_QualityEvaluation_STSAInput5_smoke].request.txt` を追加した。
- 追加した 2 本は `RuleProfileMode` を持たず、`RuleProfileAttributes` セクションから互換ラベルへ正規化して実行される。

- `dotnet build .\ShogiTournamentSystemAnalyzer\ShogiTournamentSystemAnalyzer.csproj`

- `dotnet run --project .\ShogiTournamentSystemAnalyzer\ShogiTournamentSystemAnalyzer.csproj -- --request-file "Inputs\Smoke\quality_input_[トップ集団大きめ]_[FinalStage_Single10]_[STSAInput5_smoke].request.txt"`
- `dotnet run --project .\ShogiTournamentSystemAnalyzer\ShogiTournamentSystemAnalyzer.csproj -- --request-file "Inputs\Smoke\analysis_input_[先手8x後手8]_[Simulation_QualityEvaluation_STSAInput5_smoke].request.txt"`
