# プログラム4パート構成 第13ステップ実行計画

## 目的

第12ステップで STSA のシミュレーション向け legacy 変換を分離した。
第13ステップでは、要求ファイルチェック結果モデルを `RequestFileCheck` 本体から独立させる。

## 現状

`RequestFileCheck.cs` には、次の2つが同居している。

- `RequestFileCheckResult`
- `RequestFileCheck`

`RequestFileCheckResult` は `RequestFileCheck` だけでなく、`RequestInputFileReader` と `InputSourceConfiguration` からも使われる。
そのため、チェック処理本体とは別ファイルに置いた方が責務が読みやすい。

3. `RequestFileCheck.cs` から record 定義を削る。
4. 挙動は変えない。
