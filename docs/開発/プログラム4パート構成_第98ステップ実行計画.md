# プログラム4パート構成 第98ステップ実行計画

## 目的

第97ステップで `AnalysisRequestDispatcher` の直列実行中に `SimulationDomainResult` を保持できるようになった。

今回は、後続の大会品質評価域がその実行コンテキストを受け取れる入口を用意する。まだ品質評価入力の自動補完は行わず、挙動を変えない小さな接続準備に留める。

## 今回の小さな先行作業

- `QualityEvaluationRequestDispatcher` に `AnalysisExecutionContext` 受け取り入口を追加する。
- `QualityEvaluationDomain` に `AnalysisExecutionContext` 受け取り入口を追加し、既存入口は互換 wrapper として維持する。
- `AnalysisRequestDispatcher` は品質評価ステップ実行時に直列実行中の context を渡す。
- 品質評価 mainline の入力、出力、実行オプションの扱いはまだ変えない。

## 次の実装候補

1. 品質評価要求が入力を省略した場合に、直前の `SimulationDomainResult` から評価入力を作る変換を検討する。
2. `FinalRankingDomain` に、順位行生成または出力の入口をさらに寄せる。
3. `StandardSimulationMainline` / `FinalStageSimulationMainline` から最終順位表出力を外へ出す。
4. `AnalysisRequestDispatcher` を `AnalysisDomain` 風の直列実行へ寄せる。
