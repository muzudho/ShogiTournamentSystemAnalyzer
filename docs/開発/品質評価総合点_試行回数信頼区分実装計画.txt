【品質評価総合点 試行回数と信頼区分】実装計画

目的
====
品質評価サマリーレポートの［総合点］に、必ず試行回数と信頼区分を添える。

総合点は大会ルールの品質を比較しやすくする指標だが、シミュレーション試行回数が少ない場合は値が大きく揺れる。
そのため、総合点そのものには試行回数を混ぜず、総合点の横に「この点数をどの程度の重みで見てよいか」を示す信頼区分を出す。


基本方針
========
1. 総合点の計算式は変えない
   - 試行回数を総合点へ直接加点・減点しない。
   - 大会ルール品質と計算の確からしさを分けて表示する。

2. 総合点には必ず試行回数を添える
   - Markdown、Summary CSV、Sweep CSV、コンソール出力に SimulationCount を出す。
   - 直列実行の品質評価でも、品質評価ステップで実際に使った SimulationCount を表示する。

3. 試行回数が少ない総合点は参考記録として扱う
   - 参考記録でも総合点は出す。
   - ただし比較やランキングの主材料にすると危ないことをラベルで示す。

4. 本評価は 20000 回以上にする
   - 10000 回台でも比較には使えるが、丸めや確率指標のブレを考えると本評価の線引きは 20000 回以上が妥当。


信頼区分
========
初期案:

- 1 以上 199 以下
  - ラベル: 参考記録
  - 用途: smoke、動作確認、入力形式確認
  - isReferenceRecord: true
  - isOfficialEvaluation: false

- 200 以上 1999 以下
  - ラベル: 軽量確認
  - 用途: ざっくり傾向を見る
  - isReferenceRecord: false
  - isOfficialEvaluation: false

- 2000 以上 19999 以下
  - ラベル: 比較用
  - 用途: ルール案同士の比較、次に試す条件の選定
  - isReferenceRecord: false
  - isOfficialEvaluation: false

- 20000 以上
  - ラベル: 本評価
  - 用途: レポート本文、結論、採用候補の判断
  - isReferenceRecord: false
  - isOfficialEvaluation: true

SimulationCount が 0 以下または未解決の場合は、既存の入力検証で弾くか、信頼区分を Unknown にする。
通常の品質評価では SimulationCount は必須または既定値補完済みなので、MVP では Unknown を出さない方針でよい。

==========

- SimulationCount=1 は 参考記録。
- SimulationCount=199 は 参考記録。
- SimulationCount=200 は 軽量確認。
- SimulationCount=1999 は 軽量確認。
- SimulationCount=2000 は 比較用。
- SimulationCount=19999 は 比較用。
- SimulationCount=20000 は 本評価。

出力確認:

- Summary CSV に simulationCount / scoreReliabilityLabel / isReferenceRecord / isOfficialEvaluation が出る。
- Sweep CSV に同じ項目が出る。
- SimulationCount=10 の smoke は 参考記録になる。
- SimulationCount=20000 以上の要求は 本評価になる。

退行確認:

- 既存の品質評価 smoke が通る。
- 総合点の数値自体は、信頼区分追加前後で変わらない。


注意点
======
1. 信頼区分は総合点の一部ではない
   - 58044 点の参考記録と 58044 点の本評価は、点数は同じでも信頼性が違う。
   - 比較表では点数だけでなく信頼区分も必ず見る。

2. 参考記録を隠さない
   - smoke や小さな実験でも総合点は見たい。
   - ただし、参考記録ラベルを付けて、結論に使いすぎないようにする。

3. 閾値は将来調整可能にする余地を残す
   - MVP では固定値でよい。
   - 将来、要求ファイルで ReliabilityRule を指定する拡張はあり得る。
   - ただし最初から可変にすると比較基準がぶれるので、まずは固定値を推奨する。


実装しないこと
==============
今回の計画では、次は対象外にする。

- 試行回数による総合点の補正
- 信頼区分の要求ファイル指定
- 信頼区分ごとの自動再実行
- 乱数 seed 違いの分散推定
- 信頼区間や標準誤差の計算

これらは、試行回数表示と信頼区分ラベルが安定してから検討する。


結論
====
総合点には必ず試行回数と信頼区分を添える。

総合点は大会ルール品質、信頼区分は計算の確からしさとして分けて扱う。
200回未満は参考記録、200回以上は軽量確認、2000回以上は比較用、20000回以上は本評価とする。

この方針なら、SimulationCount=10 の smoke でも総合点を楽しく見られる一方で、結論に使う評価は 20000 回以上へ自然に誘導できる。
