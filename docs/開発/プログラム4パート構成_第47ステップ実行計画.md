# プログラム4パート構成 第47ステップ実行計画

## 目的

`Application/Paths/ReportOutputPathBuilder.cs` にまとまっていた出力パス生成処理を、分析の 4 大境界へ分けて配置する。

出力先は境界データの保存場所と対応するため、FinalRanking / TournamentFinalState / TournamentQualityReport の境界フォルダーに配置する。

## 対象

- `Application/Paths/ReportOutputPathBuilder.cs`
- `Application/Execution/Analysis/Boundaries/ReportOutputPathBuilder.Core.cs`
- `Application/Execution/Analysis/Boundaries/FinalRanking/FinalRankingReportOutputPathBuilder.cs`
- `Application/Execution/Analysis/Boundaries/TournamentFinalState/TournamentFinalStateReportOutputPathBuilder.cs`
- `Application/Execution/Analysis/Boundaries/TournamentQualityReport/TournamentQualityReportOutputPathBuilder.cs`

## 作業

1. `ReportOutputPathBuilder` を `partial` class にする。
2. 共通のパス組み立てだけ `Boundaries` 直下へ置く。
3. 境界ごとの出力パス生成メソッドを各境界フォルダーへ分ける。
4. namespace は今回は維持し、呼び出し側の挙動を変えない。
5. 元の `Application/Paths/ReportOutputPathBuilder.cs` は削除する。

## 確認

- `dotnet build .\ShogiTournamentSystemAnalyzer\ShogiTournamentSystemAnalyzer.csproj`
- 空ルールの要求ファイル入力スモーク
- 手動入力保存のスモーク

## 今回は触らないこと

- namespace の大規模変更
- 出力先ディレクトリー仕様の変更
- ファイル名生成ルールの変更
