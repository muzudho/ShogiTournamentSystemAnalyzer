# プログラム4パート構成 第102ステップ実行計画

## 目的

TournamentFramework シミュレーションの出力責務を、シミュレーション主線から `FinalRankingDomain` へ寄せる。

## 現状

- 標準/本戦シミュレーションでは、最終順位表 CSV / Markdown の書き出し責務を `FinalRankingDomain` 側へ移している。
- TournamentFramework シミュレーションでは、aggregate 順位表、representative 順位表、representative 大会最終状態の複数ファイル出力が `SimulationTournamentFrameworkMode` に残っている。

## 今回の小さな作業

- `FinalRankingDomain.WriteTournamentFrameworkSimulationOutputs(...)` を追加する。
- aggregate 順位表 CSV / Markdown の生成と書き出しを移す。
- representative 順位表 CSV / Markdown の生成と書き出しを移す。
- representative 大会最終状態 CSV / Markdown の生成と書き出しを移す。
- `SimulationTournamentFrameworkMode` は計算、表示、出力入口呼び出しに絞る。

## 変えないこと

- TournamentFramework の計算ロジック。
- 出力ファイルの内容とリンク関係。
- `STSAInput/4` の仕様。

## 検証

- `dotnet build .\ShogiTournamentSystemAnalyzer\ShogiTournamentSystemAnalyzer.csproj`
- `git diff --check`
- TournamentFramework smoke:
  - `dotnet run --project .\ShogiTournamentSystemAnalyzer\ShogiTournamentSystemAnalyzer.csproj -- --request-file "Inputs/Smoke/simulation_input_[大会進行フレームワーク_最小]_[TournamentFramework_STSAInput4_smoke].request.txt"`
