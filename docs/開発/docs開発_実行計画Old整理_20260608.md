# docs/開発 実行計画Old整理 実行計画

## 目的

`docs/開発` 直下に多数並んでいる過去の numbered 実行計画を `実行計画Old` フォルダーへ移し、開発メモの入口を読みやすくする。

## 対象

- 移動対象
  - `docs/開発/プログラム4パート構成_第*ステップ実行計画.md`
- 直下に残すもの
  - `README.md`
  - `プログラム4パート構成_配置マップ.md`
  - 現在の整理計画や個別の調査メモ、復旧メモ、実装メモ

## 手順

1. `docs/開発/実行計画Old` を作成する。
2. 移動対象の numbered 実行計画を `docs/開発/実行計画Old` へ移す。
3. `docs/開発/実行計画Old/README.md` を作成し、保管対象と参照方法を書く。
4. `docs/開発/README.md` を短い入口に直し、過去の実行計画は `実行計画Old` を見るよう案内する。
5. `git status --short` とファイル数確認で移動結果を確認する。

## 完了条件

- `docs/開発` 直下の numbered 実行計画がなくなっている。
- `docs/開発/実行計画Old` に numbered 実行計画が移っている。
- `docs/開発/README.md` から `実行計画Old` へ辿れる。
