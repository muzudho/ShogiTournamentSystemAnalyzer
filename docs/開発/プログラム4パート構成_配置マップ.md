# プログラム4パート構成 配置マップ

## 作業方針

この作業では、先に `Application/Execution` 配下へライフサイクル上のフォルダーを切り、コードをそのフォルダーへ配置しながら進める。

メソッド単位の切り出しだけを続けると全体の進捗が見えにくくなるため、以後は「どのパートへ置いたか」を優先して見える化する。

## 現在のフォルダー

| フォルダー | ライフサイクル上の意味 | 配置候補 | 配置済み |
| --- | --- | --- | --- |
| `BeforeRequestFileCheck` | 要求ファイルチェック前 | コマンドライン引数確認、入力ファイル指定の有無、入力導線の初期分岐 | 第29ステップで引数読み取り、第36ステップで `InputSourceConfiguration.cs` を配置 |
| `RequestFileCheck` | 要求ファイルチェック | 要求ファイル存在確認、読み取り、形式判定 | 第28ステップでチェック本体、第34ステップで要求ファイル読み取り・STSA変換系を配置 |
| `AfterRequestFileCheck` | 要求ファイルチェック後 | チェック成功時の入力適用、失敗時の分岐 | 第30ステップで `RequestInputApplier.cs` を配置 |
| `BeforeManualInput` | 手動入力前 | 手動入力へ進む前の案内、分岐 | 未着手 |
| `ManualInput` | 手動入力 | 手動入力開始、保存確認 | 第31ステップで `ManualInput.cs` を配置 |
| `AfterManualInput` | 手動入力後 | 手動入力セッション確定、分析への接続 | 第33ステップで `ManualInputRecordingSessionStarter.cs`、第35ステップで `ManualInputRecordingTextReader.cs` を配置 |
| `BeforeRequestFileCreate` | 要求ファイル作成前 | 保存するかどうかの確認、出力先決定 | 第32ステップで `RequestFileCreatePrompt.cs` を配置 |
| `RequestFileCreate` | 要求ファイル作成 | 記録した入力から要求ファイルを書き出す処理 | 第33ステップで `RequestFileCreate.cs` を配置 |
| `AfterRequestFileCreate` | 要求ファイル作成後 | 作成完了後の通知、後片付け | 第33ステップで `RequestFileCreateCompletion.cs` / `RequestFileCreateCompletionTarget.cs` を配置 |
| `Analysis` | 分析 | 分析フロー選択、ルール選択、分析実行振り分け | 第27ステップで `Analysis.cs` / `AnalysisFlowDispatcher.cs` を配置 |

## 移動ルール

1. まず既存の責務が明確なクラスから移動する。
2. namespace は最初から大きく変えず、ビルドが通る単位で進める。
3. 移動したら、この表の配置候補と実コードの差を小さくする。
4. 1 ステップごとにビルドとスモーク確認を行う。
5. `#メモ：` や `#ここは：` で始まるファイルはユーザー用の目印として扱い、移動や編集対象から外す。

## 残る基盤

`Application/Input` には、複数ライフサイクルにまたがる入力セッション基盤を残す。

- `RequestInputSession.cs`
- `#ここは：基盤としての入力.txt`

`RequestInputSession.cs` は要求ファイル入力、手動入力、要求ファイル作成完了にまたがるため、次の設計判断まで基盤として残す。
