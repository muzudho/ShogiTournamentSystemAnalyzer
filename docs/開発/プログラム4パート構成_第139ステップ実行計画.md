# プログラム4パート構成 第139ステップ実行計画

## 目的

手入力の対象ルールプロファイル選択を、従来の属性ごとの番号入力だけでなく、`RuleProfileAttributes` の直接入力またはファイル入力からも選べるようにする。

## 現状

- 手入力のシミュレーション域と大会品質評価域は、どちらも `ConsolePromptReaders.ReadRuleProfileAttributes(...)` から `RuleProfileAttributes` を受け取る。
- `STSAInput/5` parser は `RuleProfileAttributes` セクションを既に解釈できる。
- ただし手入力では、属性を一つずつ番号メニューで選ぶ入口しかない。

## 実装手順

1. `RuleProfileAttributesTextParser` を追加し、`RuleProfileAttributes` の key=value 解析を STSAInput/5 parser と手入力で共有する。
2. `StsaInputRequestParser` の `RuleProfileAttributes` セクション解析を新 helper 呼び出しへ寄せる。
3. `ConsolePromptReaders.ReadRuleProfileAttributes(...)` に入力方法選択を追加する。
   - 属性を一つずつ入力する。
   - `RuleProfileAttributes` を key=value で直接入力する。
   - ルールプロファイルファイルから読む。
4. 手入力後も `RuleProfileAttributes.TryValidate(...)` と品質評価時の `PairingSource=ScheduledMatches` 制約を維持する。

## 完了内容

- `RuleProfileAttributesTextParser` を追加し、key=value または `#[Format] STSAInput/5` + `#[Section] RuleProfileAttributes` 形式を読めるようにした。
- `StsaInputRequestParser` の属性セクション解析を共通 helper へ寄せた。
- 手入力のルールプロファイル属性入力方法に、直接入力とファイル入力を追加した。
- smoke 生成物は実装差分から外すため削除した。
