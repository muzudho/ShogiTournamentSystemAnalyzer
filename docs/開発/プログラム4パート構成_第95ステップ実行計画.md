# プログラム4パート構成 第95ステップ実行計画

## 目的

第94ステップで `AbstractSimulationMainline.Run(...)` と派生 mainline の request 入口が `SimulationMainlineResult` を返すようになった。

今回は、その戻り値を `SimulationDomain` の内側で受け取れる入口を追加し、後続で `SimulationDomain -> FinalRankingDomain -> QualityEvaluationDomain` の直列化へ進める準備をする。

## 今回の小さな先行作業

- `SimulationDomain.TryExecute(AnalysisStepRequest step)` は既存 caller 向けの互換入口として残す。
- `SimulationDomain.TryExecute(AnalysisStepRequest step, out SimulationMainlineResult? mainlineResult)` を追加する。
- `StandardSimulationRequest` と `FinalStageSimulationRequest` は mainline の戻り値を `mainlineResult` へ返す。
- `TournamentFrameworkSimulationRequest` と `EmptySimulationRequest` は現状 mainline result を持たないため、実行成功時も `mainlineResult = null` とする。
- 既存の `AnalysisRequestDispatcher` / `SimulationRequestDispatcher` の戻り値や実行順は変えない。

## 次の実装候補

1. `SimulationRequestDispatcher` に戻り値付き入口を追加し、既存 `TryExecute(...)` は互換 wrapper にする。
2. `SimulationDomain` の戻り値を、`SimulationMainlineResult?` ではなくシミュレーション域の結果型へ包む。
3. `FinalRankingDomain` に、順位行生成または出力の入口をさらに寄せる。
4. `StandardSimulationMainline` / `FinalStageSimulationMainline` から最終順位表出力を外へ出す。
5. `AnalysisRequestDispatcher` を `AnalysisDomain` 風の直列実行へ寄せる。
