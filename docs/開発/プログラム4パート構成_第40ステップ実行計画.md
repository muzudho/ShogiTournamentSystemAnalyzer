# プログラム4パート構成 第40ステップ実行計画

## 目的

`Application/Execution/Analysis/` 配下に追加された 4大域 / 4大境界フォルダーへ、分析以降の実行ファイルを配置する。

第40ステップでは、まず責務が明確な `Application/Modes` と `Application/TournamentFramework` を 4大域側へ移す。

## 対象

### シミュレーション域

- `Application/Modes/Simulation*.cs`
- `Application/TournamentFramework/*.cs`
- `Application/Execution/Analysis/Domains/Simulation/`

### 大会品質評価フロー域

- `Application/Modes/TournamentQuality*.cs`
- `Application/Execution/Analysis/Domains/TournamentQualityEvaluator/`

## 作業

1. `Application/Execution/Analysis/Domains/Simulation/Modes/` を作る。
2. `Application/Execution/Analysis/Domains/Simulation/TournamentFramework/` を作る。
3. `Application/Execution/Analysis/Domains/TournamentQualityEvaluator/Modes/` を作る。
4. `Application/Modes/Simulation*.cs` を Simulation/Modes へ移動する。
5. `Application/TournamentFramework/*.cs` を Simulation/TournamentFramework へ移動する。
6. `Application/Modes/TournamentQuality*.cs` を TournamentQualityEvaluator/Modes へ移動する。
7. namespace は今回は維持し、外部挙動を変えない。

## 確認

- `dotnet build .\ShogiTournamentSystemAnalyzer\ShogiTournamentSystemAnalyzer.csproj`
- 空ルールの要求ファイル入力スモーク
- 手動入力保存のスモーク

## 今回は触らないこと

- `Domain/` 配下の移動
- `Infrastructure/` 配下の移動
- namespace の大規模変更
- `TournamentFramework` の解体
