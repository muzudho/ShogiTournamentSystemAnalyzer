# プログラム4パート構成 第103ステップ実行計画

## 目的

TournamentFramework シミュレーションに残っている最終順位付け寄りの処理を、さらに `FinalRankingDomain` へ寄せる。

## 現状

- 第102ステップで、TournamentFramework の aggregate 順位表 / representative 順位表 / representative 大会最終状態の書き出しは `FinalRankingDomain.WriteTournamentFrameworkSimulationOutputs(...)` へ移動済み。
- ただし `SimulationTournamentFrameworkMode` には、代表実行の `FinalRankingData.RankRows` から `RepresentativeExecutionRankRow` を作る helper が残っている。
- `FinalRankingBoundary` は空クラスで、実参照はなく、`ApplicationWorkflow` にコメント上の古い参照だけが残っている。

## 今回の小さな作業

- `FinalRankingDomain.BuildRepresentativeExecutionRankRows(...)` を追加する。
- `SimulationTournamentFrameworkMode` は代表実行順位行の生成を `FinalRankingDomain` へ委譲する。
- `SimulationTournamentFrameworkMode` のローカル `BuildRepresentativeExecutionRankRows(...)` を削除する。
- 未使用の空 `FinalRankingBoundary` と、`ApplicationWorkflow` の古いコメント参照を削除する。

## 変えないこと

- TournamentFramework の計算ロジック。
- 代表実行順位表の表示内容。
- aggregate / representative / tournament final state の出力先とファイル内容。
- `STSAInput/4` の仕様。
