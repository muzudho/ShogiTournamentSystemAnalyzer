# プログラム4パート構成 第134ステップ実行計画

## 目的

`RuleProfileAttributes` の妥当性検証を追加し、`STSAInput/5` parser が不整合な属性の組み合わせを早めに説明できるようにする。

第133ステップで標準/本戦/TournamentFramework/Empty の代表判定を `RuleProfileAttributes` に集約した。今回は、その判定が成立しない属性の組み合わせを、後続 parser の未対応扱いや不足セクションエラーへ流す前に止める。

## 実施内容

1. `RuleProfileAttributes.TryValidate(...)` を追加する。
   - `ScheduledMatches` は `PairingSource=ScheduledMatches` を必須にする。
   - `UsesFinalStageGrouping=Off` の標準形式では、本戦用属性を `Off` にする。
   - `FinalStageGrouped` は `UsesFinalStageGrouping=On` と `PairingSource=ScheduledMatches` を必須にする。
   - `TournamentFramework` は `PairingSource=TournamentFramework` とし、本戦用属性と `HasReferenceMatches` を `Off` にする。
   - `Empty` は `PairingSource=None` とし、本戦用属性と `HasReferenceMatches` を `Off` にする。
2. `StsaInputRequestParser` の `STSAInput/5` 属性セクション読み取り直後に検証を呼ぶ。
3. `STSAInput/5` 仕様へ妥当性検証の規則を追記する。

## 変えないこと

- `STSAInput/4` の互換ラベル読み取り。
- `RuleProfileAttributes` のキー構造と writer 出力形式。
- 計算ロジック、CSV / Markdown 出力内容。

## 実施結果

- `RuleProfileAttributes.TryValidate(...)` を追加した。
- `STSAInput/5` parser は、属性セクションを読んだ直後に組み合わせを検証し、不整合時はセクション名と理由を含む `OperationCanceledException` を出すようになった。
- `STSAInput/5` 仕様に属性の組み合わせ規則を追記した。
