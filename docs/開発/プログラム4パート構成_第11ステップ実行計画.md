# プログラム4パート構成 第11ステップ実行計画

## 目的

第10ステップで `RequestInputFileReader` をファイル読み取りと形式選択に近づけた。
第11ステップでは、`StsaInputLegacyConverter` の中から品質評価向け STSA 変換を専用クラスへ分離する。

## 現状

`StsaInputLegacyConverter` は STSAInput/2 / STSAInput/3 の共通入口と、複数モードの legacy 入力列変換を持っている。
特に品質評価の通常ルール / 本戦ルール変換は行数が多く、他の変換と混ざると見通しが悪い。

## 今回やること

1. `Application/Input/StsaQualityEvaluationLegacyConverter.cs` を追加する。
2. `ConvertStsaQualityEvaluationFinalStage` を移す。
3. `ConvertStsaQualityEvaluationStandard` を移す。
4. `StsaInputLegacyConverter` は品質評価の場合に専用 converter を呼ぶだけにする。
5. 変換仕様や出力文字列は変えない。

## 今回やらないこと

- 大会進行フレームワーク変換の分離
- 空ルール変換の分離
- 品質評価変換内の重複整理
- STSAInput/2 と STSAInput/3 の converter 分割

## 完了条件

- 品質評価向け STSA 変換が `StsaQualityEvaluationLegacyConverter` に分離されている。
- `StsaInputLegacyConverter` が dispatch 役に近づいている。
