# プログラム4パート構成 第6ステップ実行計画

## 目的

第5ステップで入力ファイル読み取りと変換を `RequestInputFileReader` に分離した。
第6ステップでは、その中から STSA section parse の共通処理を `StsaInputSectionParser` へ切り出す。

## 現状

`RequestInputFileReader` は次の2種類の処理を持っている。

- STSAInput/2 / STSAInput/3 を既存対話入力列へ変換する処理
- `#[Section]` / `#[EndSection]` や `key=value` を読む構文処理

構文処理は変換先のモードに依存しないため、独立させやすい。

## 今回やること

1. `Application/Input/StsaInputSectionParser.cs` を追加する。
2. `ParseStsaInputSections`、`ParseSectionKeyValues`、section/meta 取得補助を移す。
3. `RequestInputFileReader` は `using static` で parser の補助関数を使う。
4. 変換仕様やエラーメッセージは変えない。

## 今回やらないこと

- STSAInput/2 と STSAInput/3 の converter 分割
- Meta キー定義の型化
- 未知 section / 未知 key の警告対応
- parser の単体テスト追加

## 完了条件

- STSA section parse が `StsaInputSectionParser` に分離されている。
- `RequestInputFileReader` は変換処理中心になっている。
- `dotnet build` が成功する。
- `--input-file` 経由の空ルール smoke 実行が成功する。
