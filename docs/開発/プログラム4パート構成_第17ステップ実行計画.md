# プログラム4パート構成 第17ステップ実行計画

## 目的

第16ステップで `--input-file` 引数読み取りの結果をモデル化した。
第17ステップでは、`ManualInput` に残っている要求ファイル作成の質問処理を分離する。

## 現状

`ManualInput.Start()` は次の2つを担当している。

- 手動入力セッションを開始する
- 今回の入力を要求ファイルとして保存するかを質問する

後者は `RequestFileCreate` に関係する UI なので、`ManualInput` から切り出す。

## 今回やること

1. `Application/Input/RequestFileCreatePrompt.cs` を追加する。
2. `ManualInput.ReadRequestFileCreatePath` を移す。
3. `ManualInput.Start()` は `RequestFileCreatePrompt.ReadOutputPath()` を呼ぶ。
4. 表示文言と挙動は変えない。

## 今回やらないこと

- `RequestFileCreate` の保存形式変更
- 手動入力記録 reader の移動
- 要求ファイルテンプレート対応

## 完了条件

- 要求ファイル作成の質問処理が `RequestFileCreatePrompt` に分離されている。
- `ManualInput` が手動入力セッション開始に集中している。
- `dotnet build` が成功する。
- 手動入力の保存なし / 保存あり smoke が成功する。
