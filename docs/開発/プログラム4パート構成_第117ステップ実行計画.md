# プログラム4パート構成 第117ステップ実行計画

## 目的

`TournamentQualityEvaluationOutputOptions.RuleProfileMode` を出力用途専用のプロファイルへ置き換える。

品質評価の出力オプションに必要なのは、STSAInput/4 互換ラベルそのものではなく、出力ログや既定値生成で「標準相当」か「本戦相当」かを保持すること。`RuleProfileMode` は互換入力・互換出力のラベルとして残し、品質評価の出力オプション内部では小さな `TournamentQualityEvaluationOutputProfile` を使う。

## 実施内容

1. `TournamentQualityEvaluationOutputProfile` を追加する。
   - 値は `Standard` / `FinalStage` の2つに限定する。
   - `TournamentFramework` / `Empty` は品質評価出力では未対応のままにする。
2. `TournamentQualityEvaluationOutputOptions` の保持値を `RuleProfileMode` から `OutputProfile` へ変更する。
3. 依頼ログ STSAInput/4 の `RuleProfileMode` 出力に必要な箇所だけ、`GetCompatibilityRuleProfileMode()` で互換ラベルへ戻す。
4. `StsaInput4RequestParser` の品質評価 output options 生成を `TournamentQualityEvaluationOutputProfile` 指定へ変更する。
5. 手入力側の `TournamentQualityEvaluationOutputCoordinator` は、ルール定義から出力プロファイルを解決する。

## 変えないこと

- STSAInput/4 の `RuleProfileMode` 入出力互換。
- 品質評価 CSV / Markdown の内容。
- 品質評価の計算ロジック。
- `RuleProfileMode` enum 自体の削除。

## 実施結果

- `TournamentQualityEvaluationOutputProfile` を追加した。
- `TournamentQualityEvaluationOutputOptions` は `OutputProfile` を保持し、互換ログ出力時だけ `GetCompatibilityRuleProfileMode()` で `RuleProfileMode` へ戻す形にした。
- `TournamentQualityEvaluationOutputCoordinator` の `ResolveRuleProfileMode(...)` を `ResolveOutputProfile(...)` へ置き換えた。
- `StsaInput4RequestParser.ParseQualityEvaluationOutputOptions(...)` は `RuleProfileMode` ではなく `TournamentQualityEvaluationOutputProfile` を受け取るようにした。
