# プログラム4パート構成 第72ステップ実行計画

## 目的

`STSAInput/4` の `Simulation + Empty` と `Simulation + TournamentFramework` を `AnalysisRequest` 直通ルートで実行する。

## 実装範囲

- `StsaInput4RequestParser` は、単一ステップの `Simulation + Empty` / `Simulation + TournamentFramework` を legacy fallback へ戻さない。
- `AnalysisRequest` に `EmptySimulationRequest` と `TournamentFrameworkSimulationRequest` を追加する。
- `AnalysisRequestDispatcher` から既存の `SimulationEmptyMode` と `SimulationTournamentFrameworkMode` を直接呼ぶ。
- `Empty` は `OutputPath` 指定時に追加プロンプトを出さない。
- `TournamentFramework` は `Inputs` セクションの CSV パスと任意の `RuleFilePath` / `RandomSeed` / `SimulationCount` を受け取る。

## 対象モード

- `AnalysisFlowSteps=Simulation` + `RuleProfileMode=Empty`
- `AnalysisFlowSteps=Simulation` + `RuleProfileMode=TournamentFramework`

## 対象外

- `AnalysisFlowSteps=Simulation,QualityEvaluation` の複数ステップ要求。
- `QualityEvaluation + TournamentFramework` / `QualityEvaluation + Empty`。
