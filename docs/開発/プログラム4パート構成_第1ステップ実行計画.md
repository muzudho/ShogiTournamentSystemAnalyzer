# プログラム4パート構成 第1ステップ実行計画

## 目的

ライフサイクル案の「プログラム4パート構成」に向けて、最初の安全な一歩として入力導線の責務を整理する。

今回の対象は次の4パートのうち、入口の配線である。

- 要求ファイルチェック: `RequestFileCheck`
- 手動入力: `ManualInput`
- 要求ファイル作成: `RequestFileCreate`
- 分析: `Analysis`

## 現状

`Application/Input` には、すでに分離用の部品がある。

- `ManualInput.cs`
- `RequestFileCheck.cs`
- `RequestFileCreate.cs`
- `RequestInputSession.cs`

一方で、`InputSourceConfiguration.cs` の中にも次の旧実装が残っている。

- 手動入力開始処理
- 要求ファイル作成先の読み取り
- 要求ファイルチェック処理
- 手動入力記録用 `RecordingTextReader`
- 入力セッション管理用 `InputSourceConfigurationResult`

このため、まずは `InputSourceConfiguration.cs` を「入力元を選んで、分離済み部品へ渡す調停役」に寄せる。

## 今回やること

1. `InputSourceConfiguration.ConfigureInputSource()` の戻り値を `RequestInputSession` にする。
2. `--request-file` がある場合は `RequestFileCheck.TryApply(inputFilePath, ApplyInputFile)` を呼ぶ。
3. 要求ファイルチェック成功時は `RequestInputSession.FromRequestFile()` を返す。
4. 要求ファイルチェック失敗時は `ManualInput.Start()` へ切り替える。
5. `--request-file` がない場合も `ManualInput.Start()` を呼ぶ。
6. `InputSourceConfiguration.cs` 内の旧手動入力処理、旧要求ファイル作成処理、旧入力セッション、旧記録 reader を削る。

## 今回やらないこと

- `STSAInput/2` / `STSAInput/3` の変換仕様変更
- 要求ファイルチェックの検証項目追加
- `RequestFileCreate` のテンプレート出力化
- `Analysis` のクラス分離
- 既存入力ファイルの移行

## 完了条件

- `dotnet build` が成功する。
- `InputSourceConfiguration.cs` が入力元選択と入力ファイル変換に集中している。
- 手動入力の開始は `ManualInput.Start()` に集約されている。
- 要求ファイルチェックの表示と例外処理は `RequestFileCheck.TryApply()` に集約されている。
- 要求ファイル作成の保存処理は `RequestInputSession` と `RequestFileCreate` に集約されている。

## 次の候補

第1ステップ後は、次のどちらかへ進む。

- `RequestFileCheck` を「ファイルを適用する」ではなく「検査結果を返す」形へ近づける。
- `Analysis` を `Program.cs` から `Application/Execution` 側へ分離する。
