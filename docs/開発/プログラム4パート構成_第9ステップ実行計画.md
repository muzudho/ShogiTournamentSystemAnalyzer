# プログラム4パート構成 第9ステップ実行計画

## 目的

第8ステップで legacy 入力列 helper を `LegacyInputLineBuilder` に分離した。
第9ステップでは、`RequestInputFileReader` に残っている STSAInput/2 / STSAInput/3 から legacy 入力列への変換本体を `StsaInputLegacyConverter` に分離する。

## 現状

`RequestInputFileReader` はまだ次の処理を持っている。

- ファイル存在確認と読み取り
- STSAInput/2 / STSAInput/3 の判定
- legacy 入力ファイルのコメント除去
- STSA から legacy 入力列への変換本体

このうち STSA 変換本体は大きく、reader の責務を重くしている。

## 今回やること

1. `Application/Input/StsaInputLegacyConverter.cs` を追加する。
2. `ConvertStsaInput2ToLegacyInput`、`ConvertStsaInput3ToLegacyInput`、各モード別 converter を移す。
3. `RequestInputFileReader` は STSA 判定後に converter を呼ぶだけにする。
4. 変換仕様や出力文字列は変えない。

## 今回やらないこと

- STSAInput/2 と STSAInput/3 の converter 分割
- 各モード converter のクラス分割
- legacy 入力ファイルの処理分離
- テスト追加

## 完了条件

- STSA 変換本体が `StsaInputLegacyConverter` に分離されている。
- `RequestInputFileReader` がファイル読み取りと形式判定中心になる。
- `dotnet build` が成功する。
- `--input-file` 経由の空ルール smoke 実行が成功する。
