# プログラム4パート構成 第118ステップ実行計画

## 目的

`STSAInput/5` として、互換ラベル `RuleProfileMode` ではなく `RuleProfileAttributes` セクションを持つ要求ファイル形式を追加する。

第114ステップ以降、内部実行は `RuleProfileAttributes` を正として扱う方向へ寄せている。`STSAInput/4` は互換形式として維持し、`STSAInput/5` では入力面からも大分類ラベルを外す。

## 実施内容

1. 要求ファイルチェックで `#[Format] STSAInput/5` を検出する。
2. `STSAInput/5` の `RuleProfileAttributes` セクションを読む。
   - 単一ステップは `RuleProfileAttributes`。
   - 複数ステップは `Step.Simulation.RuleProfileAttributes` / `Step.QualityEvaluation.RuleProfileAttributes`。
3. 読み取った属性を既存互換ラベルへ一度戻し、既存 `STSAInput/4` parser に渡せる形へ正規化する。
4. `STSAInput/5` が直通 parser に乗らない場合、legacy 入力として誤処理せず明示エラーにする。
5. 代表 smoke として標準品質評価の `STSAInput/5` 入力を追加する。

## 変えないこと

- `STSAInput/4` の `RuleProfileMode` 入出力互換。
- Writer の既定出力形式。手入力保存と依頼ログは引き続き `STSAInput/4`。
- 品質評価・シミュレーションの計算ロジック。
- `RuleProfileMode` enum 自体の削除。

## 実施結果

- `RequestInputFormatDetector` / `RequestFileCheckWorkflow` が `STSAInput/5` を検出するようにした。
- `StsaInputValueParser` に `RuleProfileAttributes` 用の値 parser を追加した。
- `StsaInput4RequestParser` に `STSAInput/5` 正規化入口を追加し、属性セクションから互換 `RuleProfileMode` を挿入したうえで既存 parser を再利用する形にした。
- `ApplicationTournamentUser.TryConvertToLegacyInputText(...)` は `STSAInput/5` を legacy fallback へ流さない。
- `Inputs/Smoke/quality_input_[先手8x後手8]_[Neutral_Single10]_[STSAInput5_smoke].request.txt` を追加した。

## 検証

- `dotnet build .\ShogiTournamentSystemAnalyzer\ShogiTournamentSystemAnalyzer.csproj`
- STSAInput/5 Standard 品質評価 smoke: `Inputs/Smoke/quality_input_[先手8x後手8]_[Neutral_Single10]_[STSAInput5_smoke].request.txt`
