# プログラム4パート構成 第26ステップ実行計画

## 目的

`ApplicationRequestRun` の中にある「入力元を設定する処理」と「入力セッションの中で分析を実行して完了する処理」を分ける。

`ApplicationRequestRun` は依頼に対する入力セッションを用意する役割に寄せ、入力セッション内の分析と完了は `ApplicationRequestSessionRun` にまとめる。

## 対象

- `ShogiTournamentSystemAnalyzer/Application/Execution/ApplicationRequestRun.cs`
- `ShogiTournamentSystemAnalyzer/Application/Execution/ApplicationRequestSessionRun.cs`

## 作業

1. `ApplicationRequestSessionRun` を追加する。
2. `Analysis.Run()` と `inputSource.Complete()` を `ApplicationRequestSessionRun.Run(inputSource)` に移す。
3. `ApplicationRequestRun.Run(args)` は入力元設定後に `ApplicationRequestSessionRun.Run(inputSource)` を呼ぶ。
4. 外部挙動は変えない。

## 確認

- `dotnet build .\ShogiTournamentSystemAnalyzer\ShogiTournamentSystemAnalyzer.csproj`
- 空ルールの要求ファイル入力スモーク
- 手動入力保存のスモーク
