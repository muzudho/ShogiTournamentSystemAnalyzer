# プログラム4パート構成 第67ステップ実行計画

## 目的

`STSAInput/4` の `QualityEvaluation + FinalStage` を、legacy 入力列へ変換せず `AnalysisRequest` 直通ルートで実行する。

## 方針

- `FinalStageQualityEvaluationRequest` を追加する。
- `StsaInput4RequestParser` で `PlayersCsv` / `GroupMapCsv` / `AdditionalApexPlayersCsv` / `MatchesInput` / `ReferenceMatchesInput` を直接読む。
- 本戦用の `TournamentQualityEvaluationRuleDefinition` と `TournamentQualityEvaluationInput` を組み立てる。
- 20局超の要求は、標準品質評価と同じく `SimulationCount` がある場合だけ新ルートで受ける。
- 未対応の `STSAInput/4` は従来通り legacy fallback に残す。

## 実装内容

- `AnalysisRequest.cs`
  - `FinalStageQualityEvaluationRequest` を追加。
- `AnalysisRequestDispatcher.cs`
  - 本戦品質評価要求を `TournamentQualityEvaluationMainline.Run(...)` へ渡す分岐を追加。
- `StsaInput4RequestParser.cs`
  - `QualityEvaluation + FinalStage` 分岐を追加。
  - 本戦グループ、追加 Apex、境界救済、可変トップ8、Innov 順位補正のパースを追加。
  - 出力オプションの `RuleProfileMode` を Standard / FinalStage で切り替える。
