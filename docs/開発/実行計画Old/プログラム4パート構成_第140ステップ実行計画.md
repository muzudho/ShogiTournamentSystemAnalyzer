# プログラム4パート構成 第140ステップ実行計画

## 目的

`STSAInput/5` の複数ステップ要求で使っていた階層セクション名を、`Step` 接尾辞のセクション名へ変更する。

変更前の複数ステップ形式は（削除）。セクション parser は階層構造を持たず、実装上は文字列キーとして扱っている。

第140ステップでは、正規表現や配列表記を増やさず、ステップ名を先頭に出した `SimulationStep` / `QualityEvaluationStep` 形式へ寄せる。writer と parser は新形式を使う。

## 新しいセクション名

複数ステップ時のステップ固有設定:

```plaintext
#[Section] SimulationStep
#[Section] QualityEvaluationStep
```

複数ステップ時の属性:

```plaintext
#[Section] SimulationStep.RuleProfileAttributes
#[Section] QualityEvaluationStep.RuleProfileAttributes
```

複数ステップ時のステップ別入力と出力:

```plaintext
#[Section] SimulationStep.PlayersCsv
#[Section] SimulationStep.GroupMapCsv
#[Section] SimulationStep.AdditionalApexPlayersCsv
#[Section] SimulationStep.MatchesInput
#[Section] SimulationStep.ReferenceMatchesInput
#[Section] SimulationStep.Inputs
#[Section] SimulationStep.Output

#[Section] QualityEvaluationStep.PlayersCsv
#[Section] QualityEvaluationStep.GroupMapCsv
#[Section] QualityEvaluationStep.AdditionalApexPlayersCsv
#[Section] QualityEvaluationStep.MatchesInput
#[Section] QualityEvaluationStep.ReferenceMatchesInput
#[Section] QualityEvaluationStep.Output
```

単一ステップ時の `RuleProfileAttributes` / `PlayersCsv` / `MatchesInput` / `Output` は現状維持する。

## 実施内容

1. `StsaInputRequestParser` にステップセクション名 helper を追加する。
   - `FormatStepName(...)` は `Simulation` / `QualityEvaluation` を返す既存仕様のまま維持する。
   - 新 helper は `SimulationStep` / `QualityEvaluationStep` を返す。
2. 複数ステップ parser の必須セクション名を新形式へ変更する。
   - （削除） -> `SimulationStep`
   - （削除） -> `QualityEvaluationStep`
   - （削除） -> `*Step.RuleProfileAttributes`
3. 複数ステップの body section prefix を新形式へ変更する。
   - （削除） -> `SimulationStep.PlayersCsv`
   - （削除） -> `QualityEvaluationStep.Output`
4. （削除）
5. `StsaInputRequestWriter` の `STSAInput/5` 複数ステップ出力を新形式へ変更する。
   - `STSAInput/4` writer の互換 `RuleProfileMode` 出力も、複数ステップの section prefix は新形式へ合わせるかを確認する。
   - 方針は「複数ステップ section 名の整理」であり、属性形式だけに限定しない。
6. `STSAInput/5 仕様` を更新する。
   - 複数ステップ例を `SimulationStep` / `QualityEvaluationStep` 形式へ差し替える。
   - 廃止済みの旧形式は書かない旨を追記する。
7. 既存 smoke 入力を新形式へ更新する。
   - リポジトリ内の代表 `STSAInput/5` 複数ステップ要求を新形式へ寄せる。
   - （削除）

## 変えないこと

- `AnalysisFlowSteps=Simulation,QualityEvaluation` の値。
- 単一ステップ要求の section 名。
- `RuleProfileAttributes` のキー構造と検証規則。
- 計算ロジック、入力 CSV の列、出力 CSV / Markdown の内容。
- `AnalysisRequest` / `AnalysisStepRequest` の内部構造。
