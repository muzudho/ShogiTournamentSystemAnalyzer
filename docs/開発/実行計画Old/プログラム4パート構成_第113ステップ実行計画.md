# プログラム4パート構成 第113ステップ実行計画

## 目的

第112ステップ後に `SimulationTournamentFrameworkModeExecution` に残っている、ルールセット組み立てと実行サマリー表示を小さな協調クラスへ分ける。

## 現状

- TournamentFramework の入力読込、設定作成、ルールセット組み立て、集計計算、最終順位付け結果作成、サマリー表示、出力呼び出しが `SimulationTournamentFrameworkModeExecution` に並んでいる。
- 近似/厳密計算は第112ステップで `TournamentFrameworkSimulationCalculator` へ移動済み。
- 実行サマリー表示は、最終順位付け結果と集計結果の値を読むだけで、主フローに残す必然性が薄い。

## 実施内容

1. `TournamentFrameworkRuleSetFactory` を追加し、順位ルール選択と `TournamentFrameworkRuleSet` 生成を移す。
2. `TournamentFrameworkExecutionSummaryPrinter` を追加し、集計結果・代表実行・DSL 情報のコンソール表示を移す。
3. `SimulationTournamentFrameworkModeExecution` は、入力、設定、計算、最終順位付け、表示/出力連携が順に見える形へ寄せる。

## 変えないこと

- TournamentFramework の計算ロジック。
- ルールセットの中身。
- コンソール表示の文言。
- 出力 CSV / Markdown の内容。
