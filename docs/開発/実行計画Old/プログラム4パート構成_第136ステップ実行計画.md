# プログラム4パート構成 第136ステップ実行計画

## 目的

`STSAInput/5` の不整合な `RuleProfileAttributes` が、期待した説明付きで parser 段階に失敗することを smoke で固定する。

第134ステップで属性の妥当性検証を追加した。今回は、失敗する要求ファイルを通常の .NET 入口から確認できるようにし、後続の parser 整理でエラー文言が意図せず曖昧にならないようにする。

## 実施内容

1. `--smoke-request-parse-fails` を追加する。
   - 指定した要求ファイルを `StsaInputRequestParser` へ渡す。
   - `OperationCanceledException` が出た場合だけ smoke 成功にする。
   - `--contains` 指定時は、例外メッセージに期待文字列が含まれることも確認する。
2. 不整合属性の代表 smoke 入力を追加する。
   - `SimulationShape=ScheduledMatches`
   - `UsesFinalStageGrouping=Off`
   - `UsesAdditionalApexPlacement=On`
3. `ApplicationWorkflow` の開発用入口に新 smoke を接続する。
4. 開発 README と再開メモを更新する。

## 変えないこと

- 通常の要求ファイル実行時の例外捕捉。
- `STSAInput/5` の parser / writer 形式。
- 属性検証規則そのもの。
- 計算ロジック、出力 CSV / Markdown。

## 実施結果

- `RequestParseFailureSmoke` を追加し、失敗を期待する parser smoke を実行できるようにした。
- 不整合属性 smoke `Inputs/Smoke/invalid_input_[STSAInput5_RuleProfileAttributes_mismatch_smoke].request.txt` を追加した。
- `ApplicationWorkflow` から `--smoke-request-parse-fails` を呼べるようにした。

## 検証

- `dotnet build .\ShogiTournamentSystemAnalyzer\ShogiTournamentSystemAnalyzer.csproj`

- `dotnet run --project .\ShogiTournamentSystemAnalyzer\ShogiTournamentSystemAnalyzer.csproj -- --request-file "Inputs\Smoke\quality_input_[先手8x後手8]_[Neutral_Single10]_[STSAInput5_smoke].request.txt"`
