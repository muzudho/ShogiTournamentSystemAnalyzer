# プログラム4パート構成 第59ステップ実行計画

## 目的

手動入力セッション開始の中で、保存する場合と保存しない場合の分岐を読みやすくする。

また、`Application/Execution` を外した後もヘッダーコメントに残っていた `実行` 階層表記を、現在の `Application` 直下構成に合わせる。

## 対象

- `Application/ManualInput/ManualInputSessionStarter.cs`
- `Application/**/*.cs` のヘッダーコメント

## 作業

1. `ManualInputSessionStarter.Start()` の分岐を、保存なしと録音セッション開始の2択として明示する。
2. 保存しない場合のセッション作成を `StartWithoutRecording()` に切り出す。
3. ヘッダーコメントの `［アプリケーション　＞　実行　＞　...］` を現在の階層へ合わせる。
4. 挙動は変えず、読みやすさの整理だけに限定する。

## 確認

- `dotnet build .\ShogiTournamentSystemAnalyzer\ShogiTournamentSystemAnalyzer.csproj`
- 空ルールの要求ファイル入力スモーク
- 手動入力保存のスモーク
