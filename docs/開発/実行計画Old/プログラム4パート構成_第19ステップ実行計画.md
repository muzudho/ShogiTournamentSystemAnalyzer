# プログラム4パート構成 第19ステップ実行計画

## 目的

手動入力を記録して、分析後に要求ファイルとして保存する完了処理を `RequestInputSession` から分離する。

`RequestInputSession` は入力セッションの状態管理と `Console.In` の復元に寄せ、要求ファイル作成の実行は専用クラスへ移す。

## 対象

- `ShogiTournamentSystemAnalyzer/Application/Input/RequestInputSession.cs`
- `ShogiTournamentSystemAnalyzer/Application/Input/RequestFileCreateCompletion.cs`

## 作業

1. `RequestFileCreateCompletion` を追加し、記録済み入力行を要求ファイルへ書き出す処理を担当させる。
2. `RequestInputSession.CompleteRequestFileCreate()` から `RequestFileCreate.Write()` の直接呼び出しをなくす。
3. 既存の手動入力保存フローの外部挙動は変えない。

## 確認

- `dotnet build .\ShogiTournamentSystemAnalyzer\ShogiTournamentSystemAnalyzer.csproj`
- 手動入力を標準入力リダイレクトで流し、要求ファイル作成まで通ることを確認する。

## 今回は触らないこと

- 要求ファイルの書式変更
- `RequestFileCreate` の内部実装変更
- 入力フォーマット判定やレガシー変換の追加変更
