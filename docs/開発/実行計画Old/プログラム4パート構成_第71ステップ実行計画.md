# プログラム4パート構成 第71ステップ実行計画

## 目的

`STSAInput/4` の 20局超要求で `SimulationCount` が未指定の場合も、既定値を採用して `AnalysisRequest` 直通ルートで実行する。

## 実装範囲

- `StsaInput4RequestParser` は、20局超かつ `SimulationCount` 未指定の単一ステップ要求を legacy fallback へ戻さない。
- 20局超のシミュレーションと品質評価では、未指定時に既定試行回数 `200000` を `AnalysisRequest` へ設定する。
- 20局以下は従来どおり厳密計算のままにし、`SimulationCount` を補わない。
- 既に `SimulationCount` が指定されている要求は、その指定値を優先する。

## 対象モード

- `Simulation + Standard`
- `Simulation + FinalStage`
- `QualityEvaluation + Standard`
- `QualityEvaluation + FinalStage`

## 対象外

- `AnalysisFlowSteps=Simulation,QualityEvaluation` の複数ステップ要求。
- `Simulation + Empty` / `Simulation + TournamentFramework` の直通ルート化。
