# プログラム4パート構成 第50ステップ実行計画

## 目的

旧分類の `Application.Input` namespace を、ライフサイクルのフォルダー構成に合わせて分割する。

第48ステップで `Application` 直下へフォルダーを引き上げたため、第50ステップでは入力導線まわりの namespace もフォルダー名に合わせる。

## 対象

- `Application/BeforeRequestFileCheck/`
- `Application/RequestFileCheck/`
- `Application/AfterRequestFileCheck/`
- `Application/ManualInput/`
- `Application/AfterManualInput/`
- `Application/BeforeRequestFileCreate/`
- `Application/RequestFileCreate/`
- `Application/AfterRequestFileCreate/`
- `Application/Shared/RequestInputSession.cs`

## 作業

1. 各フォルダーの `Application.Input` namespace をフォルダー単位の namespace へ変更する。
2. フォルダーをまたぐ参照に `using` を追加する。
3. `RequestFileCheck` 内の static using を新 namespace へ更新する。
4. 挙動は変えず、namespace 整理だけに限定する。
