# プログラム4パート構成 第49ステップ実行計画

## 目的

第48ステップで `Application/Execution` を外したため、実行ルートの namespace から `Execution` を外してフォルダー構成と合わせる。

また、空フォルダー維持に不要になった `.gitkeep` を削除し、空フォルダーだけに `.gitkeep` を残す。

## 対象

- `ShogiTournamentSystemAnalyzer.Application.Execution` namespace / using
- `Application` 配下の `.gitkeep`

## 作業

1. 中身が空でないフォルダー直下の `.gitkeep` を削除する。
2. `ShogiTournamentSystemAnalyzer.Application.Execution` を `ShogiTournamentSystemAnalyzer.Application` へ置換する。
3. namespace の細分化は次ステップ以降に分け、今回は `Execution` 階層の除去に限定する。

## 確認

- `dotnet build .\ShogiTournamentSystemAnalyzer\ShogiTournamentSystemAnalyzer.csproj`
- 空ルールの要求ファイル入力スモーク
- 手動入力保存のスモーク

## 今回は触らないこと

- `Application.Input` / `Application.Modes` など旧分類 namespace の全面変更
- ライフサイクル内の処理順序変更
- 空フォルダー維持に必要な `.gitkeep` の削除
