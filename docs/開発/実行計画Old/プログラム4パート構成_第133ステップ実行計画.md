# プログラム4パート構成 第133ステップ実行計画

## 目的

`RuleProfileAttributes` の標準/本戦/TournamentFramework/Empty 判定を共有 helper へ寄せ、parser / legacy converter / 手入力 reader 間の重複判定を減らす。

STSAInput/5 の属性持ち回りが広がったため、各入口が同じ属性解釈を個別に書き続けると、新しい属性や互換条件を足したときに分岐差分が出やすい。まず既存挙動を変えずに、代表的な判定だけを `RuleProfileAttributes` 側へ移す。

## 実施内容

1. `RuleProfileAttributes` に次の判定プロパティを追加する。
   - `IsStandardScheduledProfile`
   - `IsFinalStageScheduledProfile`
   - `IsTournamentFrameworkProfile`
   - `IsEmptyProfile`
2. `StsaInputRequestParser` のステップ種別判定を上記プロパティへ置き換える。
3. `StsaInputLegacyConverter` の品質評価可否、legacy 変換先、手入力プロンプト番号判定を上記プロパティへ置き換える。
4. `ManualSimulationRequestReader` と `SimulationScenarioFactory` の分岐も同じ helper を使う。

## 変えないこと

- `RuleProfileAttributes` のデータ構造と既定値。
- `RuleProfileMode` 互換ラベルの意味。
- STSAInput/4 / STSAInput/5 の入出力形式。
- 計算ロジック、CSV / Markdown 出力内容。

## 実施結果

- `RuleProfileAttributes` に代表プロファイル判定を追加した。
- STSAInput parser、legacy converter、手入力シミュレーション reader、simulation scenario factory の同形判定を共有 helper へ置き換えた。
- parser / legacy converter に残っていたローカル `IsStandardScheduledProfile(...)` / `IsFinalStageScheduledProfile(...)` は削除した。
