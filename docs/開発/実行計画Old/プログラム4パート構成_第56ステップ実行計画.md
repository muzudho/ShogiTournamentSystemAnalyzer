# プログラム4パート構成 第56ステップ実行計画

## 目的

Workflow クラスの責務コメントを、ライフサイクル図の直訳ではなく責務・抽象単位に合わせる。

`ApplicationWorkflow` から詳細分岐を説明しようとすると、1 行の委譲と大きなフロー図の粒度が合わなくなるため、各 Workflow には自分が束ねる責務だけを書く。

## 対象

- `Application/ApplicationWorkflow.cs`
- `Application/RequestWorkflow.cs`
- `Application/RequestSessionWorkflow.cs`
- `Application/BeforeRequestFileCheck/InputSourceConfiguration.cs`

## 作業

1. `ApplicationWorkflow` のコメントを最上位ワークフローの説明に絞る。
2. `RequestWorkflow` のコメントを入力セッション準備の責務に絞る。
3. `RequestSessionWorkflow` のコメントを分析とセッション完了の責務に絞る。
4. `inputSource` 変数名を、実体に合わせて `inputSession` へ変更する。
5. `InputSourceConfiguration` のコメントを入力セッション設定に合わせる。
