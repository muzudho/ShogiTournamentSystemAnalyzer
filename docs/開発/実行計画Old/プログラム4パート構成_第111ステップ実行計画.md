# プログラム4パート構成 第111ステップ実行計画

## 目的

［分析］を［シミュレーション域］、［最終順位付け域］、［大会品質評価域］へ分ける作業の続きとして、残っている古い境界クラスと、別の域に残った責務を外へ出す。

## 現状

第110ステップまでで、直列実行の入口と TournamentFramework の最終順位付け結果の受け渡しはかなり整理済み。

- `SimulationDomain` はシミュレーション域の入口になっている。
- `FinalRankingDomain` は最終順位行生成、最終順位表の表示、最終順位表の出力入口を持っている。
- `QualityEvaluationDomain` は大会品質評価域の入口になっている。
- `AnalysisRequestDispatcher` は `Simulation,QualityEvaluation` の直列実行コンテキストを持ち回れる。
- TournamentFramework シミュレーション側は、最終順位付け結果を `TournamentFrameworkFinalRankingResult` として渡す形へ寄っている。

ただし、以下の残骸と混線が残っている。

- `Domain/TournamentFinalState/TournamentFinalStateBoundary.cs` は空クラスのまま残っている。
- `ApplicationWorkflow.cs` に `TournamentFinalStateBoundary` のコメント参照が残っている。
- `Application.Analysis.Boundaries` の `BuildTournamentFinalStateBoundaryData(...)` と `BuildFinalRankingBoundaryData(...)` は未使用に見える。
- `FinalRankingDomain.WriteTournamentFrameworkSimulationOutputs(...)` が、代表実行の大会最終状態 CSV / Markdown 出力まで担当している。
- `SimulationTournamentFrameworkModeExecution.cs` に、TournamentFramework の place probability 集計と順位ルール別の順位確率計算が残っている。

## 実施内容

1. 空の `TournamentFinalStateBoundary` を削除する。
2. `ApplicationWorkflow.cs` の古い境界コメントを現状の 3 域構成に合わせて更新する。
3. 未使用の `BuildTournamentFinalStateBoundaryData(...)` と `BuildFinalRankingBoundaryData(...)` を削除できるか確認し、参照がなければ削除する。
4. TournamentFramework の代表実行大会最終状態出力を、最終順位付け域から外す。
   - 候補: `TournamentFinalStateDomain` または `TournamentFinalStateOutputCoordinator` を追加する。
   - `FinalRankingDomain.WriteTournamentFrameworkSimulationOutputs(...)` は、代表順位表と aggregate 最終順位表の出力へ寄せる。
   - 大会最終状態 CSV / Markdown のファイル名解決と writer 呼び出しは、大会最終状態側へ移す。
5. TournamentFramework の place probability 集計を、最終順位付け域へ移す。
   - `AccumulateTournamentFrameworkPlaceProbabilities(...)`
   - `AccumulateTournamentFrameworkNeutralPlaces(...)`
   - `AccumulateTournamentFrameworkTwillPlaces(...)`
   - `BuildTournamentFrameworkPointsByPlayerId(...)`
6. `SimulationTournamentFrameworkMode` は、実行結果と集計に必要な raw 値を最終順位付け域へ渡すだけに近づける。

## 判断メモ

- `TournamentFinalStateData` 自体は境界データなので残す。
- `TournamentFinalStateData` が `Domain.Simulation.TournamentMatchRecord` に依存している点は、今回は無理にほどかない。大会フレームワークの対局記録型がまだシミュレーション域にあるため、別ステップで扱う方が安全。
- 品質評価側の `BoundaryDataBuilders.BuildTournamentQualityReportBoundaryData(...)` と `BuildTournamentQualitySweepReportBoundaryData(...)` は現役参照があるので、第111ステップでは削除対象にしない。
- `ReportOutputPathBuilder` は複数境界で共有されているため、今回は利用箇所の移動に留める。命名や配置の整理は後続ステップでよい。

## 変えないこと

- TournamentFramework の計算結果。
- 出力される CSV / Markdown の内容。
- 出力ファイル名の意味。
- `STSAInput/4` の仕様。
- 標準シミュレーション、本戦シミュレーション、品質評価の実行結果。

## 実施結果

- 空の `TournamentFinalStateBoundary` を削除した。
- 未使用だった `BuildTournamentFinalStateBoundaryData(...)` と `BuildFinalRankingBoundaryData(...)` を削除した。
- `ApplicationWorkflow.cs` の古い `TournamentFinalStateBoundary` コメントを削除し、順位付け域のコメントを最終順位付け域へ合わせた。
- `TournamentFinalStateDomain` を追加し、TournamentFramework 代表実行の大会最終状態 CSV / Markdown 出力を移した。
- TournamentFramework の place probability 集計を `FinalRankingDomain.AccumulateTournamentFrameworkPlaceProbabilities(...)` へ移した。
- `SimulationTournamentFrameworkMode` は、順位確率集計を `FinalRankingDomain` へ委譲する形にした。
