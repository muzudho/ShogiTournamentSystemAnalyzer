# プログラム4パート構成 第99ステップ実行計画

## 目的

`AnalysisFlowSteps=Simulation,QualityEvaluation` で、品質評価ステップが入力データを省略した場合に、直前のシミュレーション request から品質評価入力を組み立てられるようにする。

## 今回の範囲

- `AnalysisExecutionContext` に直前シミュレーション request を保持する。
- `STSAInput/4` parser で、品質評価入力セクションが丸ごと無い場合だけ deferred 品質評価 request を作る。
- `QualityEvaluationDomain` で deferred request を解決し、標準/本戦シミュレーション request から `TournamentQualityEvaluationInput` を組み立てる。
- 20局超で品質評価 `SimulationCount` が未指定の場合は、従来どおり既定値 `200000` を補う。

## 対応条件

- 標準品質評価の入力省略は、直前ステップが `StandardSimulationRequest` の場合だけ対応する。
- 本戦品質評価の入力省略は、直前ステップが `FinalStageSimulationRequest` の場合だけ対応する。
- `TournamentFramework` / `Empty` のシミュレーション結果からは、品質評価入力を復元しない。
- 入力セクションが一部だけ欠けている場合は、従来どおり要求ファイル不備として扱う。

## 検証

- `dotnet run --project .\ShogiTournamentSystemAnalyzer\ShogiTournamentSystemAnalyzer.csproj -- --request-file Inputs/Smoke/analysis_input_[先手1x後手1]_[Simulation_QualityEvaluation_InferredQualityInput_STSAInput4_smoke].request.txt`
