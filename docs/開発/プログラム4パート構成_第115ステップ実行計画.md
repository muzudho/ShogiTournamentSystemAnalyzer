# プログラム4パート構成 第115ステップ実行計画

## 目的

`AnalysisRequest.RuleProfileMode` を内部実行の正から外す。

要求全体の大分類ラベルは STSAInput/4 互換表示・旧フロー橋渡し用としてだけ扱い、直通ルートの実行判断と要求ファイル書き出しは各 `AnalysisStepRequest` から導出した `RuleProfileAttributes` / 互換ラベルを使う。

## 実施内容

1. `AnalysisRequest` のトップレベル `RuleProfileMode` 保持を削除する。
2. 互換ラベルが必要な箇所向けに、`Steps[0]` から `GetCompatibilityRuleProfileMode()` で導出する入口を追加する。
3. `StsaInput4RequestParser` と `ManualAnalysisRequestReader` の `AnalysisRequest` 生成を、新しい形へ更新する。
4. `ApplicationTournamentUser` と `ProgramConsoleGuide` は、直通要求がある場合に `AnalysisRequest` から互換ラベルを導出して表示・旧フロー互換値へ使う。

## 変えないこと

- STSAInput/4 の `RuleProfileMode` 入力互換。
- 手入力 UI の大分類選択。
- legacy fallback 側の `TournamentUserDomainResult.RuleProfileMode`。
- `StsaInput4RequestWriter` の互換 `RuleProfileMode` 出力。

## 実施結果

- `AnalysisRequest` の positional parameter から `RuleProfileMode` を削除した。
- `AnalysisRequest.GetCompatibilityRuleProfileMode()` を追加し、互換ラベルは先頭 step の属性から導出する形へ寄せた。
- STSAInput/4 parser の複数ステップ処理から、要求全体用の `ruleProfileModes` 保持を削除した。
- 手入力から作る `AnalysisRequest` も、step list だけを持つ形へ更新した。
- コンソールの選択主線表示は、直通要求がある場合 `AnalysisRequest` 由来の互換ラベルを使うようにした。
