# プログラム4パート構成 第22ステップ実行計画

## 目的

`RequestInputSession` の完了 API を、要求ファイル作成専用の名前からセッション全体の完了を表す名前へ整える。

セッション内部では、完了対象がある場合だけ要求ファイル作成を実行する。呼び出し元の `Program` は、具体的な完了内容ではなく「入力セッションを完了する」と読めるようにする。

## 対象

- `ShogiTournamentSystemAnalyzer/Program.cs`
- `ShogiTournamentSystemAnalyzer/Application/Input/RequestInputSession.cs`

## 作業

1. `RequestInputSession.CompleteRequestFileCreate()` を `Complete()` に変更する。
2. `Program.cs` の呼び出しを `inputSource.Complete()` に変更する。
3. コメントもセッション完了として読める表現に寄せる。
4. 外部挙動は変えない。
