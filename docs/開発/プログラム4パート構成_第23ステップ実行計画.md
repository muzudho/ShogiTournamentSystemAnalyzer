# プログラム4パート構成 第23ステップ実行計画

## 目的

`Program.cs` に残っている主実行フローを `Application.Execution` へ移し、プログラム入口を薄くする。

`Program.cs` はコンソール出力エンコーディング設定と例外表示を持つ入口に寄せ、アプリケーションとしての実行順序は `ApplicationRun` にまとめる。

## 対象

- `ShogiTournamentSystemAnalyzer/Program.cs`
- `ShogiTournamentSystemAnalyzer/Application/Execution/ApplicationRun.cs`

## 作業

1. `ApplicationRun` を追加する。
2. アプリケーション予算開始、導入表示、入力元設定、分析実行、入力セッション完了を `ApplicationRun.Run(args)` に移す。
3. `Program.Main()` は出力エンコーディング設定、`ApplicationRun.Run(args)` 呼び出し、`OperationCanceledException` の表示に絞る。
4. 外部挙動は変えない。

## 確認

- `dotnet build .\ShogiTournamentSystemAnalyzer\ShogiTournamentSystemAnalyzer.csproj`
- 空ルールの要求ファイル入力スモーク
- 手動入力保存のスモーク

## 今回は触らないこと

- 例外処理の種類追加
