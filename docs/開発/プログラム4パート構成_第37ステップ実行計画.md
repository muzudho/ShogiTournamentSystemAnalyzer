# プログラム4パート構成 第37ステップ実行計画

## 目的

複数パートにまたがる入力セッション基盤を `Application/Execution/Shared/` へ配置する。

`RequestInputSession` は、要求ファイル入力、手動入力、要求ファイル作成完了の各パートから参照される。単一のライフサイクル節点へ置くと責務が偏って見えるため、例外的に `Shared` に置く。

## 対象

- `ShogiTournamentSystemAnalyzer/Application/Input/RequestInputSession.cs`
- `ShogiTournamentSystemAnalyzer/Application/Execution/Shared/`

## 作業

1. 配置マップを更新する。
2. `RequestInputSession.cs` を `Application/Execution/Shared/` へ移動する。
3. namespace は今回は維持し、外部挙動を変えない。
4. `#メモ：` や `#ここは：` で始まるユーザー用メモは触らない。

## 確認

- `dotnet build .\ShogiTournamentSystemAnalyzer\ShogiTournamentSystemAnalyzer.csproj`
- 空ルールの要求ファイル入力スモーク
- 手動入力保存のスモーク
