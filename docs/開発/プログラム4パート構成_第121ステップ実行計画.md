# プログラム4パート構成 第121ステップ実行計画

## 目的

手入力・旧フロー側の品質評価主線に残っている `RuleProfileMode` 依存を減らす。

`RuleProfileMode` は入力互換ラベルとして残しつつ、品質評価の実行判断は `RuleProfileAttributes` の `UsesFinalStageGrouping` と `PairingSource` を正にする。

## 実施内容

1. `TournamentQualityEvaluationInputReader.TryReadQualityEvaluationRuleDefinition(...)` に `RuleProfileAttributes` 版入口を追加する。
2. 既存の `RuleProfileMode` 版入口は互換 wrapper として属性へ変換する。
3. `TournamentQualityEvaluationMainline.Run(...)` に `RuleProfileAttributes` 版入口を追加する。
4. `TournamentQualityEvaluationMode` と手入力の品質評価 request reader は、互換ラベルへ戻さず属性を渡す。
5. 標準品質評価か本戦品質評価かの request 型選択は `UsesFinalStageGrouping` で判定する。

## 変えないこと

- `STSAInput/4` parser の互換 `RuleProfileMode` 分岐。
- 手入力 UI の表示文言。
- 品質評価の計算ロジック、出力 CSV / Markdown の内容。

## 実施結果

- 品質評価の対話入力主線は `RuleProfileAttributes` を受け取ってルール定義を作る形へ寄せた。
- `RuleProfileMode` 版の `TournamentQualityEvaluationInputReader` / `TournamentQualityEvaluationMainline` 入口は互換 wrapper として残した。
- `TournamentQualityEvaluationMode` と `ManualQualityEvaluationRequestReader` は、互換ラベルへ戻さず属性を渡すよう変更した。

- 手入力の対象ルール選択を属性直接入力・ルールプロファイルファイル選択へ拡張する。
