## 暫定運用

- 1スレッドのログが 500 KiB を超えたら、`docs/続きはここから.md` を更新して新スレッドへ移る候補にする。
- 650 KiB を超えたら、作業継続よりも新スレッド移行を優先する。
- 長いファイル確認は `Select-Object -First`、`rg`、`git diff --stat` などで要約し、必要箇所だけ読む。
- 大きいコマンド出力が必要な場合は、件数・サイズ・パスだけ表示し、全文出力を避ける。
- バッククォートを含む Markdown/docs 全体への広い検索は避ける。必要なファイル、見出し、行数に絞ってから読む。
- `rg --files` はリポジトリ全体で出すと 40 KiB 級になり得るため、必要なディレクトリや拡張子に絞る。
- `docs/続きはここから.md` は長くなっているため、全文読み取りではなく `Select-Object -First` や `rg -n` で必要箇所だけ読む。

## 新スレッド移行について

Codex 側から自動で新しいスレッドを開く操作はできない。ユーザーが新スレッドを開き、最初に `docs/続きはここから.md` を読ませる運用にする。

移行前に Codex が行うこと:

1. `docs/続きはここから.md` に、完了作業、未完了作業、検証結果、注意点を短く追記する。
2. `git status --short` と `git diff --cached --stat` を確認する。
3. 必要ならこの調査メモに、その時点のセッションログサイズを追記する。
4. ユーザーへ「新スレッドへ移るのがよい」と明示する。

新スレッド冒頭でユーザーが伝える文例:

```text
前スレッドからの続きだぜ。まず docs/続きはここから.md を読んで、現在の差分と次の作業を確認してくれだぜ。
```
