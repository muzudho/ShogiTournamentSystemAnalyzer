## まず全体像

アプリ本体は `ShogiTournamentSystemAnalyzer` フォルダーにあります。

- `Program.cs`
  - エントリーポイントです
  - 起動メッセージ、入力ソース設定、モード分岐だけを持ちます
  - 「最初にどこから始まるか」を追いたいときの入口です
- `Presentation`
  - コンソール表示や入力の責務です
- `Application`
  - モード進行、入力適用、出力先決定、検証などの**アプリ進行役**です
- `Infrastructure`
  - CSV 出力や入力パースなど、外側とのやりとりです
- `Domain`
  - ルール、計算、モデル、列挙型などの**中核ロジック**です
- `Examples`
  - 人が読むためのサンプル CSV とメモです
- `Inputs`
  - `--input-file` で流す実行用入力ファイルです
- `Output`
  - 通常実行時の出力先です

## 起点になるファイル

### `ShogiTournamentSystemAnalyzer/Program.cs`
最初に読むならここです。

- `ConfigureInputSource(args)`
  - `--input-file` を使う流れへ入ります
- `ReadMode()`
  - モード選択へ入ります
- `RunStandardMode()`
  - 通常モードへ入ります
- `RunFinalStageMode()`
  - 本戦専用モードへ入ります
- `RunQualityEvaluationMode()`
  - 品質評価モードへ入ります

「どのファイルがどこから呼ばれているか」を大づかみしたいときは、まずここから追うのが早いです。

## 最初に読む順

初見で追うなら、次の順がおすすめです。

1. `ShogiTournamentSystemAnalyzer/Program.cs`
   - 起動から各モードへどう分岐するかを見る
2. `Application/Modes/*Mode.cs`
   - そのモードの骨格を見る
	  - `SimulationStandardMode.cs` / `SimulationFinalStageMode.cs` / `TournamentQualityEvaluationMode.cs`
3. `Application/Modes/*Input.cs` / `*Execution.cs` / `*Output.cs`
   - 入力、実行、出力のどこを直すかで追う
4. `Application/Validation` と `Application/Reporting`
   - 検証やレポート行生成を見たいときに読む
5. `Infrastructure/Parsing` と `Infrastructure/DataFiles`
   - 入出力フォーマットを直したいときに読む
 6. `Domain/Simulation` / `Domain/TournamentRule` / `Domain/TournamentQualityEvaluator` / `Domain/Ranking`
   - 5大域 / 6大境界 に沿った中核ロジックを直したいときに読む

「まず骨格、そのあと入力・実行・出力、最後に Domain」という順で見ると迷いにくいです。

## フォルダー別の役割

### `Presentation/Console`
コンソール入出力まわりです。

- `ConsolePrompts.cs`
  - 案内文、選択肢、プロンプト表示を直したいとき
- `ConsoleRuleSelections.cs`
  - ルール選択 UI を直したいとき
- `ConsoleInputReaders.cs`
  - コンソールからの読み取り手順を直したいとき
- `ConsoleSamples.cs`
  - サンプル表示や入力例を直したいとき
- `ConsoleResultPrinter.cs`
  - コンソールに出る結果表、品質評価サマリー表示を直したいとき

### `Application/Modes`
各モードの進行役です。現在の入口は、まず「対局シミュレーション / 品質評価」を選び、その後に「通常ルール / 本戦ルール」を選ぶ形です。

5大域 / 6大境界 で見るなら、ここは各大域の進行をつなぐ場所で、`Application/Helpers/ApplicationFlowDataMappers.cs` が 6 大境界データの組み立て口です。

- `SimulationStandardMode.cs`
	- 対局シミュレーション / 通常ルールの進行の骨格を直したいとき
- `SimulationStandardModeInput.cs`
	- 対局シミュレーション / 通常ルールの入力読取を直したいとき
- `SimulationStandardModeExecution.cs`
	- 対局シミュレーション / 通常ルールの計算実行、結果表示、CSV 出力を直したいとき
- `SimulationFinalStageMode.cs`
	- 対局シミュレーション / 本戦ルールの進行の骨格を直したいとき
- `SimulationFinalStageModeInput.cs`
	- 対局シミュレーション / 本戦ルールの入力読取や検証を直したいとき
- `SimulationFinalStageModeExecution.cs`
	- 対局シミュレーション / 本戦ルールの計算分岐、文脈表示、出力書き出しを直したいとき
- `TournamentQualityEvaluationMode.cs`
	- 品質評価 / 通常ルール・本戦ルールの進行の骨格や入力解釈を直したいとき
- `TournamentQualityEvaluationExecution.cs`
  - 品質評価の実行手順、スイープ、試行設定を直したいとき
- `TournamentQualityEvaluationOutput.cs`
  - 品質評価の表示、出力先選択、CSV 出力まわりを直したいとき

「モードの流れを変えたい」ときは、まずここを見ます。

### `Application/Validation`
入力内容や大会条件の整合性チェックです。

- `FinalStageValidators.cs`
  - 本戦 16 人条件、Apex 重複、グループ整合などの検証を直したいとき

### `Application/Reporting`
表示・レポート用の行データや集計を作る場所です。

- `ResultBuilders.cs`
		- Application 側に残る表示寄りのレポート補助を追いたいとき

### `Application/Helpers`
大域間や境界間の受け渡しを補助する場所です。

- `ApplicationFlowDataMappers.cs`
	- 6大境界（`TournamentRuleData` / `PlayerListData` / `RankingSettingsData` / `TournamentFinalStateData` / `FinalRankingData` / `TournamentQualityReportData`）の組み立て口を追いたいとき
  - 境界データ本体は `Domain/TournamentRule` / `Domain/PlayerList` / `Domain/RankingSettings` / `Domain/TournamentFinalState` / `Domain/FinalRanking` / `Domain/TournamentQualityReport` に分かれています

### `Application/Paths`
レポートの出力先やファイル名を決める場所です。

- `ReportOutputPathBuilder.cs`
  - `Output/Reports/Good` / `Bad` への振り分けや既定ファイル名を直したいとき

### `Application/Input`
入力ファイルの適用です。

- `InputSourceConfiguration.cs`
  - `--input-file` の解釈、コメント行除去、`#[Enter]` などの入力前処理を直したいとき

### `Application/Execution`
実行時間や共通の実行制御です。

- `SimulationBudget.cs`
  - シミュレーション時間制限や確率正規化を直したいとき

### `Application/Helpers`
複数モードで共有する軽い進行補助です。

- `ModeSupportHelpers.cs`
  - モード共通の補助処理を直したいとき

### `Infrastructure/DataFiles`
データファイルの入出力処理です。

- `Shared/CsvOutputHelpers.cs`
  - CSV 文字列の整形や補助を直したいとき
- `Shared/CsvSchemaCommonColumns.cs`
  - CSV の共通部列（`boundaryName` / `schemaName` / `rowType`）を直したいとき
- `FinalRanking/FinalRankingDataFileWriter.cs`
  - 通常結果表・本戦結果表・representative順位表の書き出しを直したいとき
- `TournamentFinalState/TournamentFinalStateDataFileWriter.cs`
  - 大会最終状態 CSV / Markdown の書き出しを直したいとき
- `TournamentQualityReport/TournamentQualityReportDataFileWriter.cs`
  - 品質評価サマリー・選手別・スイープ出力を直したいとき
- `TournamentFramework/TournamentFrameworkCsvParsers.cs`
  - 大会進行フレームワーク用 CSV の読み取りを直したいとき

### `Infrastructure/Parsing`
CSV や対局表入力の読み取りです。

- `InputParsers.cs`
  - 選手一覧 CSV、対局 CSV、Round/Black-White/対局記号表の読み取りを直したいとき

### `Domain/Proposer`

## 変更時のおすすめ確認順

コードを直したあと、次の順で確認すると安全です。

1. 変更した責務の近いファイルを見直す
   - 例: `Application/Modes` を直したら、対応する `Input` / `Execution` / `Output` も軽く確認する
2. ビルドする
   - まず `dotnet build` でコンパイルが通るか確認する
3. 必要なら入力ファイルで軽く動かす
   - `Inputs/Smoke` は軽い確認向きです
   - `Inputs/Bench` は単発評価向きです
   - `Inputs/Sweeps` は条件を振って確認したいとき向きです
4. 出力先や CSV を確認する
   - `Output`
   - `Output/Reports`
5. 説明書に影響があれば docs も更新する
   - `README.md`
   - `docs/README.md`
   - `docs/Manuals/README.md`
   - `docs/Manuals/実装ファイル案内.md`

## 実行用ファイルと資料の置き場

人が読むための見本やメモです。

- 詳細: `ShogiTournamentSystemAnalyzer/Examples/README.md`
