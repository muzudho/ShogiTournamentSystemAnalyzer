# 品質評価

## 概要
品質評価は、大会ルールが選手の実力をどの程度そのまま順位へ反映できているかを確認するための機能です。

単発の品質評価だけでなく、先手勝率を一定範囲で変化させる **n% スイープ実験** にも対応しています。

- 主な用途:
	- 通常ルール案の比較
  - 本戦ルール案の比較
  - `Neutral` と `Twill` の比較
  - `Apex / Innov` 分割あり・なしの比較
  - 改善案の `On / Off` 比較

品質評価は、起動後に次の 2 段階で選びます。

- 目的: `2. 品質評価`
- 対象ルール:
  - `1. 通常ルール`
  - `2. 本戦ルール`

## 基本的な考え方
品質評価では、まず **ニュートラル** を基準にします。
ニュートラルより良い大会ルールを `Good`、悪くなるルールを `Bad` として扱います。

`Apex / Innov` を `Off` にした場合は、品質評価でも `Neutral / Twill` を切り替えられます。

`Twill` の評価では、`Neutral` を基準にして次の指標がどう変わるかを見る使い方を想定しています。

- Spearman 相関
- 平均順位ずれ
- 上位8残留
- Elo1位の総合1位確率

## 評価指標
品質評価モードでは、順位分布そのものに加えて、次の品質指標を表示し、CSV と Markdown レポートを出力します。

- Spearman 相関
- 平均順位ずれ
- Elo上位8名の総合上位8位残留人数（平均）
- Elo1位の総合1位確率
- 品質評価総合点
- 最大不利益 / 最大利益

### 指標の見方
- **Spearman 相関**
  - Elo順位と最終順位の並びがどれだけ近いかを見る指標です。
  - 値が高いほど、実力順を保ちやすいと見なせます。
- **平均順位ずれ**
  - 各選手について、Elo順位と期待総合順位の差の絶対値を平均したものです。
  - 小さいほど、順位のぶれが少ないと見なせます。
- **上位8残留**
  - Elo 上位 8 名が、総合上位 8 位にどれだけ残るかの平均です。
  - 本戦進出者選抜の妥当性を見たいときに使います。
- **Elo1位の総合1位確率**
  - 最上位実力者が優勝できる確率です。
  - 最強者を正しく上に置けるかの目安になります。
- **品質評価総合点**
  - 複数の品質指標を 1 つの点数にまとめた値です。
  - 既定の満点は `100000` 点です。
  - 点数は単発評価のサマリー CSV / Markdown / コンソールと、n% スイープ実験の CSV / Markdown / コンソールに出力します。
- **最大不利益 / 最大利益**
  - どの選手が最も順位面で損・得をしているかを見るための指標です。

### 総合点の使い方
このシステムの目的は、良い大会ルールを作ることです。
品質評価総合点は、そのために大会ルール案を横並びで探すためのスコアボードとして使います。

基本的な流れは次です。

1. まず総合点が高い大会ルール案を候補にします。
2. `参考記録` や `軽量確認` の点数は、候補探しの目安として扱います。
3. 上位候補は `本評価` になる試行回数で再計算します。
4. 最後は総合点だけで決めず、内訳と大会の考え方を見て選びます。

総合点が高いほど、現在のスコアルールでは良い大会ルール案と見なせます。
ただし、総合点は唯一の正解ではありません。
同じ総合点でも、実力順の保持を重視した点なのか、本戦上位8名の保持を重視した点なのかで意味が変わります。

そのため、点が近い案や最終候補では、次の内訳を確認します。

- `Spearman`: 実力順が保たれているか
- `平均順位ずれ`: 順位がどれだけ歪んでいるか
- `上位8残留`: 強い8人を上位8位に残せているか
- `Elo1位の総合1位確率`: 最強者が勝ち切れる余地があるか

`Balanced`、`ChampionFocused`、`Top8Focused` のようにスコアルールが違う総合点同士は、直接比較しないでください。
比較するときは、同じスコアルール、同じ入力条件、同じ程度の試行回数で揃えます。

## 品質評価総合点
品質評価総合点は、次の 4 指標を 0.0 から 1.0 の範囲へ正規化し、重みを掛けて合計します。

| 指標 | 正規化の考え方 | 既定の重み |
| --- | --- | --- |
| Spearman 相関 | `(-1.0..1.0)` を `(0.0..1.0)` に変換 | `40000` |
| 平均順位ずれ | 許容値 `4.0` 以下なら高評価、許容値以上は 0 点 | `25000` |
| 上位8残留 | `8.0` 人を満点として評価 | `20000` |
| Elo1位の総合1位確率 | 確率をそのまま評価 | `15000` |

既定プリセットは `Balanced` です。用途に応じて、`ChampionFocused` または `Top8Focused` も選べます。

| プリセット | Spearman | 平均順位ずれ | 上位8残留 | Elo1位優勝 |
| --- | ---: | ---: | ---: | ---: |
| `Balanced` | `40000` | `25000` | `20000` | `15000` |
| `ChampionFocused` | `30000` | `20000` | `15000` | `35000` |
| `Top8Focused` | `30000` | `20000` | `35000` | `15000` |

`STSAInput/5` では、`QualityScoreRule` セクションでプリセットや重みを指定できます。重みを個別指定する場合は、4 項目の合計が `ScoreMax` と一致する必要があります。

## 入力

### 品質評価 / 通常ルール
1. 順位ルールセット（`Neutral / Twill`）
2. 同Elo対局時の先手勝率 (%)
3. 選手 / Player 一覧CSV
4. 対局CSV または Round/First-Second / PlayerSymbols
5. 単発評価または n% スイープ実験の選択
6. 必要に応じてシミュレーション試行回数
7. レポート出力先パスまたはフォルダーパス

### 品質評価 / 本戦ルール
品質評価 / 本戦ルールの基本入力は、対局シミュレーション / 本戦ルールとほぼ同じです。

1. 同Elo対局時の先手勝率 (%)
2. 選手 / Player 一覧CSV
3. Apex / Innov の分け方（On / Off）
4. `Off` のときだけ順位ルールセット（`Neutral / Twill`）
5. `On` のときだけグループ対応CSV
6. `On` のときだけ本戦不出場Apex一覧CSV（省略可）
7. `On` のときだけ本戦不出場Apexの扱い（Off / On）
8. `On` のときだけ境界救済戦の有無（Off / On）
9. `On` のときだけ可変定員8ルールの有無（Off / On）
10. `On` のときだけ品質評価用の Innov 比較基準順位補正（Off / On）
11. 対局CSV または Round/First-Second / PlayerSymbols
12. 参考対局CSV または Round/First-Second / PlayerSymbols（省略可）
13. 単発評価または n% スイープ実験の選択
14. 必要に応じてシミュレーション試行回数
15. レポート出力先パスまたはフォルダーパス

## 切り替え可能なルール
品質評価では、実験用に次を切り替えられます。

### 順位ルールセット
- `Neutral`: 勝ち数ベースのニュートラル順位
- `Twill`: 比較グラフと重箱表で順位を決める

品質評価 / 通常ルールでは、この切り替えが中心です。
品質評価 / 本戦ルールでは、Apex / Innov を `Off` にしたときだけ `Neutral / Twill` を切り替えます。

### Apex / Innov の分け方
- `Off`: フラットに順位づけする
- `On`: Apex / Innov を使う

### 本戦不出場Apexの扱い
- `Off`: Innov より前に順位帯を確保する（現行案）
- `On`: 総合順位へ挿入しない（改善案A）

### 境界救済戦
- `Off`: 使わない
- `On`: Apex最下位相当と Innov 最上位相当で救済戦を行う

### 可変定員8ルール
- `Off`: 定員8固定
- `On`: 本戦不出場Apex人数ぶんだけ Innov 上位を総合上位8へ引き上げる

### 品質評価用の Innov 比較基準順位補正
- `Off`: Innov も Elo 順位そのままで期待順位ずれを比べる
- `On`: 本戦不出場Apexを Innov より前へ置く制度に合わせて、Innov の**比較基準順位**を `本戦不出場Apex人数 + 1` ぶん後ろへずらして期待順位ずれを計算する

これは**品質評価の比較基準だけ**を切り替えるオプションです。実際の大会順位計算そのものは変えません。

## n% スイープ実験
品質評価では、単発評価のほかに **n% スイープ実験** を選べます。
これは先手勝率を一定範囲で振りながら、各 n% に対する品質指標の変化をまとめて CSV と Markdown レポートへ出力する機能です。

例えば、50% から 100% まで 5 刻みで評価すると、先手有利が強くなるほどルールの公平性や実力反映性がどう変わるかを比較できます。

実行用入力ファイルは、主に次に配置します。

- `Inputs/Smoke`
- `Inputs/Bench`
- `Inputs/Sweeps`

## 参考対局
対局シミュレーション / 本戦ルール と 品質評価 / 本戦ルールでは、**参考対局** を別入力で受け付けられます。

- 参考対局は表示・CSV保存されます
- **大会記録や品質評価には含めません**

大会本体と、補足として見せたい参考結果を分けたいときに使います。

## レポート運用
品質評価レポートは、必要に応じて次のように分けて管理できます。

- `Output/TournamentQualityEvaluator/TournamentQualityReport/Summary/Good`
- `Output/TournamentQualityEvaluator/TournamentQualityReport/Summary/Bad`
- `Output/TournamentQualityEvaluator/TournamentQualityReport/Players/Good`
- `Output/TournamentQualityEvaluator/TournamentQualityReport/Players/Bad`
- `Output/TournamentQualityEvaluator/TournamentQualityReport/Sweeps/Good`
- `Output/TournamentQualityEvaluator/TournamentQualityReport/Sweeps/Bad`

基準は **ニュートラル** です。
ニュートラルより良いものを `Good`、悪いものを `Bad` とし、必要なら **評価メモ** を 1 行残して運用します。

現在の出力は、数値確認用の CSV と、GitHub 上で読みやすい Markdown レポートを併用する形です。

### 命名について
品質評価の選手別 CSV は、現在は `*_quality_players.csv` を正とします。

- `*_quality_players.csv`
  - 現在の選手別品質評価 CSV 名

文書では基本的に **選手 / Player** を使うため、今後の案内は `*_quality_players` で統一します。
