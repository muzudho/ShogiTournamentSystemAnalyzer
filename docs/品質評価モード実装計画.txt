【品質評価モード】実装計画

目的
======
大会ルールが「Elo レーティングの高いソフトほど制度上の総合順位でも高くなりやすいか」を、
シミュレーション結果から定量評価できるモードを追加する。

このモードは、本戦専用モードの結果を利用して、
ルールの良し悪しを説得材料として示すための分析機能である。


このモードで答えたい問い
========================
- 高 Elo のソフトほど、制度上の総合順位でも上位になりやすいか？
- 制度的なグループ分割（Apex / Innov）は、実力順をどれだけ保っているか？
- 先手・後手の不公平を是正しつつ、実力反映性をどこまで保てているか？
- 既存ルールや単純な総当たり戦と比べて、このルールは説得力があるか？


前提
====
品質評価モードは、まず本戦専用モードの上に載せる。
すなわち、次の情報が取得できることを前提とする。

- 各参加者の Elo
- 各参加者のグループ（Apex / Innov）
- 各参加者の制度上総合順位分布
- 各参加者の期待総合順位
- 各参加者のグループ内順位分布

将来的には予選込み大会シミュレーションにも拡張可能だが、
最初は本戦専用モードと組み合わせる形で十分である。


品質評価の基本方針
==================
1. 公平性と実力反映性を分けて考える
   - 公平性: 先手後手の偏りを制度が吸収できているか
   - 実力反映性: Elo 順と最終順位がどれだけ一致しているか

2. 説明しやすい指標を優先する
   専門家向けに高度な統計指標を増やしすぎるより、
   一般に説明しやすい指標をまず出す。

3. 複数の指標で見る
   相関 1 つだけでは不十分。
   順位相関、上位保持率、逆転率、境界ペナルティなどを合わせて見る。


MVP（最小実装）
================
最初の実装では、次の指標を出せれば十分に価値が高い。

1. Elo 順位と期待総合順位の順位相関
   - Spearman の順位相関係数

2. 期待総合順位ずれ
   - 各参加者について
     期待総合順位 - Elo 順位
   - その絶対値平均（Mean Absolute Rank Error 相当）

3. 上位保持率
   - Elo 上位 8 名のうち、制度上総合上位 8 位に平均何名残るか
   - Elo 1 位が制度上総合 1 位になる確率

4. 境界ペナルティ
   - 高 Elo なのに Innov にいることで 9 位以下に押し込まれる影響
   - 逆に、低 Elo なのに Apex にいることで上位帯へ残る影響


推奨する指標一覧
================
A. 順位相関
-----------
1. Spearman 相関
   - Elo 順位と期待総合順位の相関
   - 1.0 に近いほど実力順を保っている

2. Kendall 相関（将来候補）
   - 順位対の整合性を見る
   - MVP では後回しでよい


B. 順位誤差
-----------
1. 平均順位ずれ
   - mean(|期待総合順位 - Elo順位|)

2. 最大順位ずれ
   - 最も制度の影響を受けた参加者のずれ幅

3. Elo 上位者の順位ずれ一覧
   - 特に Elo 上位 5 名や上位 8 名を個別表示


C. 上位保持率
-------------
1. Elo 上位 8 名の総合上位 8 位残留人数
2. Elo 1 位が総合 1 位になる確率
3. Elo 上位 3 名が総合上位 3 位に残る確率
4. Elo 上位 8 名が総合上位 8 位を独占する確率


D. 逆転・歪み指標
-----------------
1. 強者逆転率
   - Elo が高い参加者が、制度上の期待総合順位で低 Elo に負ける割合

2. 境界ペナルティ
   - Elo 順位が高いのに Innov に入ったことで受ける不利益
   - Elo 順位が低いのに Apex に入ったことで受ける利益

3. グループ境界の歪み
   - Apex/Innov の境界が Elo 順位のどこに来ているか
   - 境界付近の Elo を持つ参加者の順位変動の大きさ


実装対象の出力案
================
コンソールで出す要約:
- Spearman 相関
- 平均順位ずれ
- Elo 上位 8 名の総合上位 8 位残留人数（平均）
- Elo 1 位の総合 1 位確率
- 最も大きい境界ペナルティを受けた参加者

CSV で出す明細:
- participantName
- group
- originalElo
- eloRank
- expectedOverallPlace
- expectedGroupPlace
- overallPlaceDeltaFromEloRank
- overallTop1Probability
- overallTop8Probability
- groupTop1Probability
- boundaryPenaltyIndex

ルール全体の要約CSV:
- metricName
- metricValue
- note


品質評価モードの操作案
======================
起動直後のモード選択に、次を追加する。

1. 通常モード（現行の総当たり戦分析）
2. 本戦専用モード（Apex / Innov 定先戦分析）
3. 品質評価モード（本戦ルールの実力反映性評価）

品質評価モードでは、入力としては本戦専用モードとほぼ同じものを受け付ける。
ただし出力が「順位分布中心」ではなく「品質指標中心」になる。


本戦専用モードとの関係
======================
設計上は次の 2 案がある。

案A: 品質評価モードを独立モードにする
------------------------------------
利点:
- 利用者に分かりやすい
- 出力責務が明確

欠点:
- 本戦専用モードと入力処理が重複しやすい

案B: 本戦専用モードの追加出力として実装する
--------------------------------------------
利点:
- コード再利用しやすい
- 同じシミュレーション結果をそのまま使える

欠点:
- 本戦専用モードの責務が重くなる

推奨:
MVP では案Bが現実的。
まず本戦専用モードの結果計算を共通化し、
その結果から品質指標を追加計算する。
利用者向けには「品質評価モード」と見せてもよいが、
実装内部では本戦専用モードを呼び出す構造にするとよい。


必要な内部データ
================
品質評価指標を計算するには、各参加者について少なくとも次が欲しい。

- Name
- Group
- OriginalElo
- EloRank
- ExpectedOverallPlace
- ExpectedGroupPlace
- OverallPlaceProbabilities
- GroupPlaceProbabilities

また、ルール全体の集約値として次を作る。
- SpearmanCorrelation
- MeanAbsoluteRankError
- AverageTop8Retention
- EloTop1ToOverallTop1Probability
- MaxBoundaryPenalty


関数分割案
==========
- RunQualityEvaluationMode()
- BuildQualityMetrics()
- CalculateSpearmanCorrelation()
- CalculateMeanAbsoluteRankError()
- CalculateTopRetentionMetrics()
- CalculateBoundaryPenaltyMetrics()
- BuildQualityMetricRows()
- WriteQualityMetricsCsv()

本戦専用モード側で流用する関数:
- RunFinalStageModeCore()
- BuildFinalStageResultRows()
- BuildFinalStageSummary()

ポイント:
本戦専用モードの「シミュレーション実行」と「表示」を分離し、
品質評価モードは前者を再利用して後者だけ差し替える形にする。


境界ペナルティの考え方
======================
本ルール特有の説得材料として重要なのが境界ペナルティである。

例:
- Elo 3 位なのに Innov に入ったため、制度上 9 位以下にしかなれない
- Elo 11 位なのに Apex に入ったため、制度上 8 位以内へ入りやすい

MVP では単純に、
  境界ペナルティ = 期待総合順位 - Elo順位
の大きさで見てもよい。

ただし将来は、
- グループ固定がなければ何位相当だったか
- 通常総当たり戦ならどうだったか
との比較に拡張すると説得力が増す。


比較対象
========
品質評価は、単独の数値でも useful だが、比較相手があると強い。
将来は次と比較できるとよい。

1. 通常の総当たり戦
2. ランダムに近い対戦表
3. 現行大会ルールの近似
4. 白黒対抗ルール案

MVP では比較機能までは入れなくてもよいが、
出力形式は将来比較しやすいようにしておく。


段階的な実装順
==============
第1段階: 本戦専用モードの結果計算を共通化
- シミュレーション本体と表示を分離
- 品質評価から再利用できるようにする

第2段階: Elo 順位の算出
- 入力参加者を Elo 順に並べる
- 同 Elo の扱い方針を決める

第3段階: MVP 指標追加
- Spearman 相関
- 平均順位ずれ
- 上位保持率
- 境界ペナルティ

第4段階: 表示とCSV出力
- コンソール要約
- 指標CSV
- 参加者別CSV

第5段階: ドキュメント整備
- README に品質評価モードを追記
- 具体例を Examples / Docs に追加


テスト観点
==========
1. 指標計算
- 完全に Elo 順通りなら相関が高くなる
- 明らかに逆転が多ければ相関が下がる
- 順位ずれが大きいケースで MAE が大きくなる

2. ルール特有の確認
- Innov の高 Elo 参加者が 9 位以下へ押し込まれると、境界ペナルティが大きく出る
- Apex の低 Elo 参加者が上位帯へ残ると、境界ペナルティが負方向に出る

3. 退行確認
- 本戦専用モードの結果が変わっていない
- 通常モードのビルドが通る


おすすめの実装順（結論）
==========================
1. 本戦専用モードを先に実装する
2. 本戦専用モードの結果計算を共通化する
3. その上に品質評価モードを追加する
4. まずは Spearman 相関、平均順位ずれ、上位保持率、境界ペナルティの 4 指標に絞る

これで、
「このルールは先後の不公平を吸収しつつ、どの程度 Elo 順を保てるか」
を数字で説明できるようになる。

以上。
