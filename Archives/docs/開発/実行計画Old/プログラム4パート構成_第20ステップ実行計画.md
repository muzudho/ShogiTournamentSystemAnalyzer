# プログラム4パート構成 第20ステップ実行計画

## 目的

`RequestInputSession` が持っている「要求ファイル作成のためのパス」と「記録用 TextReader」をひとまとまりの完了対象として扱う。

これにより、`RequestInputSession` の null フィールドの組み合わせを減らし、入力セッション本体は「完了処理があれば実行する」という見通しに寄せる。

## 対象

- `ShogiTournamentSystemAnalyzer/Application/Input/RequestInputSession.cs`
- `ShogiTournamentSystemAnalyzer/Application/Input/RequestFileCreateCompletionTarget.cs`
- `ShogiTournamentSystemAnalyzer/Application/Input/RequestFileCreateCompletion.cs`

## 作業

1. `RequestFileCreateCompletionTarget` を追加し、要求ファイル作成パスと記録入力を保持する。
2. `RequestInputSession` の要求ファイル作成用フィールドを `RequestFileCreateCompletionTarget?` にまとめる。
3. `RequestFileCreateCompletion.Complete()` は完了対象を受け取る形にする。
4. 外部挙動は変えない。

## 確認

- `dotnet build .\ShogiTournamentSystemAnalyzer\ShogiTournamentSystemAnalyzer.csproj`
- 空ルールの要求ファイル入力スモーク
- 手動入力保存のスモーク

## 今回は触らないこと

- 入力ファイル読み取りや STSA 変換処理
