# プログラム4パート構成 第126ステップ実行計画

## 目的

リポジトリに置く代表要求ファイルと、アプリが新規に出力する要求ファイル系ログを `STSAInput/5` へ寄せる。

`STSAInput/4` の読み取り互換は維持するが、新しく保存・生成・案内する形式は `RuleProfileAttributes` セクションを持つ `STSAInput/5` を本命にする。

## 実施内容

1. 品質評価の依頼ログテンプレートを `STSAInput/5` へ更新する。
   - `RuleProfileMode` は出力せず、`RuleProfileAttributes` セクションを出す。
   - 既存のログ責務は広げず、現行のログ用ひな形としての出力を維持する。
2. `--smoke-request-writer-roundtrip` の既定生成形式を `STSAInput/5` にする。
   - `--format STSAInput/4` 指定時だけ互換 writer を使う。
   - 入力ファイルの形式は `#[Format]` から判定し、`STSAInput/4` / `STSAInput/5` の両方を round-trip できるようにする。
3. writer が deferred 品質評価 request を `STSAInput/5` として出せるようにする。
   - `Simulation,QualityEvaluation` で品質評価入力を省略する smoke を最新版形式へ移せるようにする。
4. `Inputs/Smoke` の `STSAInput/4` smoke を `STSAInput/5` へ変換する。
   - `STSAInput/4` ファイルは通常の代表 smoke から外す。
   - `STSAInput/4` 読み取り互換は parser と明示指定 smoke で維持する。
5. docs の現行仕様説明を更新する。

## 変えないこと

- `STSAInput/4` の読み取り互換。
- `STSAInput/4` writer の互換出力入口。
- 過去ステップの開発記録。
- 計算ロジック、順位計算、品質評価指標、CSV / Markdown の実データ内容。

## 実施結果

- 品質評価の依頼ログテンプレートを `#[Format] STSAInput/5` と `RuleProfileAttributes` セクション出力へ更新した。
- writer round-trip smoke の既定生成形式を `STSAInput/5` に変更し、入力形式判定も `STSAInput/4` / `STSAInput/5` 両対応へ変更した。
- `StsaInput4RequestWriter` に deferred 標準/本戦品質評価 request の出力対応を追加した。
- `Inputs/Smoke` の `STSAInput4_smoke` 要求ファイルを `STSAInput5_smoke` へ変換し、旧4ファイルを削除した。
