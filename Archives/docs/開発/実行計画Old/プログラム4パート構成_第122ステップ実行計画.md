# プログラム4パート構成 第122ステップ実行計画

## 目的

旧対話シミュレーションの scenario / execution plan に残っていた `RuleProfileMode` 依存を減らす。

`RuleProfileMode` は入力互換ラベルとして残し、シミュレーション側のシナリオ表現は `RuleProfileAttributes` を持つ形へ寄せる。

## 実施内容

1. `ISimulationScenario` の `RuleProfileMode` プロパティを `RuleProfileAttributes` へ置き換える。
2. `StandardSimulationScenario` / `FinalStageSimulationScenario` は、それぞれの既定属性を直接保持する。
3. `SimulationExecutionPlan` から未使用の `RuleProfileMode` を削除する。
4. `SimulationScenarioFactory.Create(RuleProfileMode)` の未使用互換入口を削除する。
5. UseCases 配下に `RuleProfileMode` が残っていないことを確認する。

## 変えないこと

- 旧対話シミュレーションの入力順や表示文言。
- 標準 / 本戦シミュレーションの計算ロジック。
- `STSAInput/4` / `STSAInput/5` の互換入力。
- 出力 CSV / Markdown の内容。

## 実施結果

- `ISimulationScenario` は `RuleProfileAttributes` を公開する形へ変更した。
- 標準 / 本戦 scenario の計画生成から互換 `RuleProfileMode` を外した。
- `SimulationExecutionPlan` は実行ラベルと実行 delegate だけを持つ形へ整理した。
- `SimulationScenarioFactory` の未使用 `RuleProfileMode` overload を削除した。
- `Application.Analysis.Domains.Simulation.UseCases` 配下の `RuleProfileMode` 残存検索はヒットなしになった。
