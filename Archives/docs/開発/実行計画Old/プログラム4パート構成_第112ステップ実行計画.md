# プログラム4パート構成 第112ステップ実行計画

## 目的

第111ステップ後に `SimulationTournamentFrameworkModeExecution` に残っていた TournamentFramework の計算責務を、シミュレーション域内の大会進行フレームワーク用クラスへ分離する。

## 現状

- `SimulationTournamentFrameworkModeExecution` は、入力 CSV 読込、DSL 読込、実行設定、ルールセット作成、集計、最終順位付け域への受け渡し、表示・出力呼び出しを同じファイルで扱っている。
- TournamentFramework の近似シミュレーション、20局以下の厳密計算、標準相当の対局結果 resolver も同じファイルに同居している。
- place probability の集計自体は第111ステップで `FinalRankingDomain.AccumulateTournamentFrameworkPlaceProbabilities(...)` へ移動済み。

## 実施内容

1. TournamentFramework の集計結果型を `TournamentFrameworkSimulationAggregate` として `Simulation/TournamentFramework` 配下へ移す。
2. 近似シミュレーションと厳密計算を `TournamentFrameworkSimulationCalculator` へ移す。
3. 未確定対局を標準相当の勝率で解決する `StandardLikeMatchResultResolver` を独立ファイルへ移す。
4. `SimulationTournamentFrameworkModeExecution` は、入力・設定・ルールセット・エンジン作成と、計算結果の表示/出力連携に寄せる。

## 変えないこと

- TournamentFramework の計算ロジック。
- 厳密計算と近似シミュレーションの切り替え条件。
- 出力 CSV / Markdown の内容とファイル名の意味。
- `STSAInput/4` の仕様。

## 実施結果

- `TournamentFrameworkSimulationAggregate.cs` を追加した。
- `TournamentFrameworkSimulationCalculator.cs` を追加し、既定シミュレーション回数、厳密計算閾値、近似ループ、厳密探索を移した。
- `StandardLikeMatchResultResolver.cs` を追加し、未確定対局の結果解決を移した。
- `SimulationTournamentFrameworkModeExecution.cs` から計算 helper と resolver を削除し、`TournamentFrameworkSimulationCalculator.Execute(...)` 呼び出しへ差し替えた。
