# プログラム4パート構成 第138ステップ実行計画

## 目的

`RuleProfileMode` enum を `LegacyRuleProfileMode` へ改名し、互換入力・互換出力専用の型として `Compatibility/Legacy*` 配下へ移す。

単なる型移動だけで進めると、`RuleProfileAttributes` など本筋の Domain 型が `Compatibility` に依存する。第138ステップでは、その逆依存を作らないように互換ラベル変換を先に切り分ける。

## 現状

- `RuleProfileMode` は `ShogiTournamentSystemAnalyzer/Domain/TournamentQualityEvaluator/TournamentQualityEvaluatorEnums.cs` にある。
- C# の実参照は少なく、主に次の場所に残っている。
  - `Domain/TournamentQualityEvaluator/RuleProfileAttributes.cs`
  - `Application/RequestFileCheck/StsaInputValueParser.cs`
  - `Application/RequestParsing/StsaInputRequestParser.cs`
  - `Application/RequestFileWrite/StsaInputRequestWriter.cs`
  - `Compatibility/LegacyRequestFile/StsaInputLegacyConverter.cs`
- `RuleProfileAttributes.FromCompatibilityLabel(...)` と `RuleProfileAttributes.ToCompatibilityLabel()` が、Domain から互換 enum へ直接触っている。
- STSAInput/4 の `RuleProfileMode=...` は読み取り互換・書き出し互換として維持する。
- STSAInput/5 と内部実行は `RuleProfileAttributes` を正とする。

## 事前に切り分ける障害

### 1. Domain から Compatibility への逆依存

`LegacyRuleProfileMode` を `Compatibility/Legacy*` 配下へ移したあと、`RuleProfileAttributes` が `LegacyRuleProfileMode` を受け返しすると、Domain が Compatibility に依存する。

対応:

- `RuleProfileAttributes.FromCompatibilityLabel(...)` を削除するか、少なくとも次の実装ステップで呼び出しをなくす。
- `RuleProfileAttributes.ToCompatibilityLabel()` を削除するか、少なくとも writer / legacy converter 側へ移す。
- 互換 enum と `RuleProfileAttributes` の相互変換は `Compatibility` または要求ファイル境界の専用 helper に置く。

候補名:

- `Compatibility/LegacyRuleProfile/LegacyRuleProfileMode.cs`
- `Compatibility/LegacyRuleProfile/LegacyRuleProfileMapper.cs`

`LegacyRuleProfileMapper` は `LegacyRuleProfileMode` と `RuleProfileAttributes` の変換を担当する。依存方向は `Compatibility -> Domain` に留める。

### 2. Application の request parser / writer が Compatibility に依存する範囲

`StsaInputRequestParser` と `StsaInputRequestWriter` は STSAInput/4 と STSAInput/5 の両方を扱う。ここに `Compatibility` using を入れると、現行 STSAInput/5 の本筋 parser も互換層に依存しているように見える。

対応:

- まずは `StsaInputRequestParser` 内の `STSAInput/4` 分岐だけが `LegacyRuleProfileMapper.ParseAttributesFromLabel(...)` を呼ぶ形にする。
- `STSAInput/5` の `RuleProfileAttributes` section parser は Compatibility を経由しない。
- writer は `BuildLines(..., useAttributeFormat: false)` の STSAInput/4 出力時だけ mapper で互換ラベルへ変換する。
- 将来さらに厳密にするなら、STSAInput/4 parser / writer を `Compatibility` 側へ分離し、Application には `AnalysisRequest` を返す facade だけ見せる。

第138ステップでは、実装量を抑えるため「STSAInput/4 の分岐だけ互換 helper を呼ぶ」までを目標にする。

### 3. RequestFileCheck の汎用値 parser が互換 enum を返す

`StsaInputValueParser.ParseRuleProfileMode(...)` は現在 `RuleProfileMode` を返している。ここをそのまま `LegacyRuleProfileMode` に変えると、`Application.RequestFileCheck` が Compatibility の型を返すことになる。

対応:

- `ParseRuleProfileMode(...)` は public/internal helper として残さない。
- 互換ラベル文字列の解釈は `LegacyRuleProfileMapper` 側へ寄せる。
- `StsaInputValueParser.ParseRuleProfileAttributesFromCompatibilityLabel(...)` も `Compatibility` 側へ移すか、互換 wrapper にする。
- `StsaInputValueParser` には `RuleProfileAttributes` section 用の値 parser だけを残す。

### 4. エラーメッセージと要求ファイル仕様の表記

外部ファイルのキー名 `RuleProfileMode` は STSAInput/4 互換仕様として残す。型名を `LegacyRuleProfileMode` に変えても、ユーザー向けエラーや出力ファイルのキー名は変えない。

対応:

- エラーメッセージは `RuleProfileMode=...` のまま維持する。
- docs/設計 の STSAInput/2-4 仕様も `RuleProfileMode` のまま維持する。
- 内部コード名だけ `LegacyRuleProfileMode` にする。

## 実装手順

1. `Compatibility/LegacyRuleProfile` フォルダーを追加する。
2. `RuleProfileMode` enum を `LegacyRuleProfileMode` に改名し、`Compatibility/LegacyRuleProfile/LegacyRuleProfileMode.cs` へ移す。
3. `LegacyRuleProfileMapper` を追加する。
   - `ParseMode(string value, string formatName)`
   - `FromCompatibilityLabel(LegacyRuleProfileMode mode, TournamentRuleSetMode rankingRuleSetMode = TournamentRuleSetMode.Neutral)`
   - `ToCompatibilityLabel(RuleProfileAttributes attributes)`
   - `FormatLabel(RuleProfileAttributes attributes)` または `ToString()` 利用方針を固定する。
4. `RuleProfileAttributes` から `RuleProfileMode` 参照を削除する。
   - `FromCompatibilityLabel(...)` を mapper へ移す。
   - `ToCompatibilityLabel()` を mapper へ移す。
5. `StsaInputValueParser` から `RuleProfileMode` 返却 helper を削除する。
6. `StsaInputRequestParser` の STSAInput/4 分岐を mapper 呼び出しへ変更する。
7. `StsaInputRequestWriter` の STSAInput/4 出力を mapper 呼び出しへ変更する。
8. `StsaInputLegacyConverter` の互換エラー表示を mapper 呼び出しへ変更する。
9. `rg -n "\bRuleProfileMode\b" ShogiTournamentSystemAnalyzer -g "*.cs"` で型名残存を確認する。
   - 文字列 `"RuleProfileMode"` は STSAInput/4 キー名として残ってよい。
   - 型名・メンバー参照としての `RuleProfileMode` は残さない。
10. `rg -n "Compatibility" ShogiTournamentSystemAnalyzer/Domain -g "*.cs"` で Domain から Compatibility への依存がないことを確認する。
