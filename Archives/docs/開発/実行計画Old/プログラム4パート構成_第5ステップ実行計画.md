# プログラム4パート構成 第5ステップ実行計画

## 目的

第4ステップで `--request-file` 引数読み取りを `RequestFileArgumentReader` に集約した。
第5ステップでは、`InputSourceConfiguration` に残っている入力ファイル読み取りと STSA/legacy 変換処理を専用クラスへ分離する。

## 現状

`InputSourceConfiguration` は入力元選択の調停役になりつつあるが、まだ次の処理を大量に持っている。

- 入力ファイル存在確認
- legacy 入力ファイルの `#[Enter]` / コメント処理
- `STSAInput/2` / `STSAInput/3` の section parse
- section から既存対話入力列への変換
- STSA 内の選択値 parse

このためファイル名と責務の対応が弱い。

## 今回やること

1. `Application/Input/RequestInputFileReader.cs` を追加する。
2. `InputSourceConfiguration.ReadInputFile()` と関連する変換関数群を `RequestInputFileReader` へ移す。
3. `InputSourceConfiguration` は `RequestFileCheck.TryRead(inputFilePath, RequestInputFileReader.Read, out ...)` を呼ぶだけにする。
4. `InputSourceConfiguration` から不要な using を削る。

## 今回やらないこと

- STSAInput の変換仕様変更
- 変換関数の細かな再設計
- STSAInput/2 と STSAInput/3 のクラス分割
- parser のテスト追加

## 完了条件

- `InputSourceConfiguration` が入力元選択と `Console.In` 適用だけを担当する。
- 入力ファイル読み取りと変換は `RequestInputFileReader` にまとまっている。
- `dotnet build` が成功する。
- `--request-file` 経由の空ルール smoke 実行が成功する。
