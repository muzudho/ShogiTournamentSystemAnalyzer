# プログラム4パート構成 第97ステップ実行計画

## 目的

第96ステップで `SimulationRequestDispatcher` が `SimulationDomainResult` を返せるようになった。

今回は、`AnalysisRequestDispatcher` の内部でシミュレーション域の結果を受け取り、後続ステップへ渡すための実行コンテキストを用意する。

## 今回の小さな先行作業

- `AnalysisExecutionContext` を追加し、直列実行中の `LastSimulationResult` を保持できるようにする。
- `AnalysisRequestDispatcher.Execute(...)` で 1 要求につき 1 つの context を作り、各ステップへ持ち回る。
- シミュレーションステップは `SimulationRequestDispatcher.TryExecute(..., out SimulationDomainResult? result)` を呼び、結果を context に保存する。
- 品質評価ステップの実行方法と外部公開 API はまだ変えない。

## 次の実装候補

1. `QualityEvaluationRequestDispatcher` / `QualityEvaluationDomain` に context 受け取り入口を追加する。
2. 品質評価要求が入力を省略した場合に、直前の `SimulationDomainResult` から評価入力を作る変換を検討する。
3. `FinalRankingDomain` に、順位行生成または出力の入口をさらに寄せる。
4. `StandardSimulationMainline` / `FinalStageSimulationMainline` から最終順位表出力を外へ出す。
5. `AnalysisRequestDispatcher` を `AnalysisDomain` 風の直列実行へ寄せる。
