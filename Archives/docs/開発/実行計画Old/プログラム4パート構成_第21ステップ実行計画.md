# プログラム4パート構成 第21ステップ実行計画

## 目的

`RequestInputSession` の生成 API を、入力元の理由ではなくセッションが持つ状態で表現する。

要求ファイル入力と、要求ファイル作成をしない手動入力は、どちらも「完了処理なし」のセッションで同じ状態になる。保存する手動入力だけが「要求ファイル作成完了対象あり」のセッションになる。

## 対象

- `ShogiTournamentSystemAnalyzer/Application/Input/RequestInputSession.cs`
- `ShogiTournamentSystemAnalyzer/Application/Input/InputSourceConfiguration.cs`
- `ShogiTournamentSystemAnalyzer/Application/Input/ManualInput.cs`
- `ShogiTournamentSystemAnalyzer/Application/Input/ManualInputRecordingSessionStarter.cs`

## 作業

1. `RequestInputSession.FromRequestFile()` と `FromManualInputWithoutRequestFileCreate()` を、状態を表す `WithoutCompletion()` に統合する。
2. `FromManualInputWithRequestFileCreate()` を、完了対象を受け取る `WithRequestFileCreateCompletion()` に置き換える。
3. `RequestFileCreateCompletionTarget` の生成は、手動入力記録セッション開始側へ寄せる。
4. 外部挙動は変えない。
