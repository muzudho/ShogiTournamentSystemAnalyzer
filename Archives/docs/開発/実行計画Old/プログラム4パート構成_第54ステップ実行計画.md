# プログラム4パート構成 第54ステップ実行計画

## 目的

`Application.Analysis.Analysis` という名前の反復を解消する。

`Analysis` フォルダーと `Analysis` クラスが同名になっていたため、分析実行を表すクラス名を `AnalysisRun` に変更して読みやすくする。

## 対象

- `Application/Analysis/Analysis.cs`
- `Application/Analysis/AnalysisRun.cs`
- `Application/ApplicationRequestSessionRun.cs`

## 作業

1. `Analysis.cs` を `AnalysisRun.cs` へリネームする。
2. `internal static class Analysis` を `internal static class AnalysisRun` へリネームする。
3. 呼び出し側の alias をやめ、通常の `using` で参照する。
4. 挙動は変えず、命名整理だけに限定する。

## 確認

- `dotnet build .\ShogiTournamentSystemAnalyzer\ShogiTournamentSystemAnalyzer.csproj`
- 空ルールの要求ファイル入力スモーク
- 手動入力保存のスモーク

## 今回は触らないこと

- `AnalysisFlowDispatcher` の責務変更
- 4 大域 / 4 大境界の再分類
